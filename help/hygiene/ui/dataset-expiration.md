# データセットの有効期限の自動化 {#dataset-expiration}

>[!CONTEXTUALHELP]
>id="platform_privacyConsole_scheduleDatasetExpiration_description"
>title="不要または期限切れの顧客レコードとデータセットの削除"
>abstract="<h2>説明</h2><p>規制への準拠に関係なく、Experience Platform データのライフサイクルを管理するために、消費者データを削除し、データセットの有効期限をスケジュールできます。 データ主体のリクエストを作成または管理するには、「データ主体のプライバシーリクエストに従う」ユースケースブロックを参照してください。</p>"

Adobe Experience Platform UIの [[!UICONTROL データライフサイクル] ワークスペース](./overview.md) を使用すると、データセットの有効期限をスケジュールできます。 データセットが有効期限に達すると、データレイク、ID サービスおよびリアルタイム顧客プロファイルは別個のプロセスを開始して、それぞれのサービスからデータセットの内容を削除します。3 つのサービスすべてからデータを削除すると、有効期限が完了とマークされます。

>[!WARNING]
>
>データセットの有効期限が切れるように設定されている場合、ダウンストリームワークフローに悪影響が及ばないよう、データをそのデータセットに取り込む可能性があるデータフローを手動で変更する必要があります。

このドキュメントでは、Platform UIでのデータセットの有効期限をスケジュールおよび自動化する方法について説明します。

>[!NOTE]
>
>データセットの有効期限では、現在、Adobe Experience Platform Edge ネットワークからデータが削除されることはありません。 ただし、データセットの有効期限が切れるように設定されている場合、データがエッジネットワーク内に残る可能性はありません。 これは、データセットの有効期限に関する 15 日間のサービスライセンス版契約が、データが破棄される前にエッジ ネットワーク内に存在する 14 日間の期間と重複しているためです。

詳細 データ ライフサイクル管理では、 [データセット 有効期限 エンドポイントを介したデータセットの削除](../api/dataset-expiration.md) および [workorder エンドポイント](../api/workorder.md)を介したプライマリ ID を使用した ID の削除 (行レベルのデータ) がサポートされています。 また、有効期限や [レコードの削除](./record-delete.md) をPlatform UIから管理 データセットすることもできます。 詳しくは、リンク先のドキュメントを参照してください。

>[!NOTE]
>
>データ ライフサイクルはバッチ削除をサポートしていません。

## データセット有効期限のスケジュール設定 {#schedule-dataset-expiration}

>[!CONTEXTUALHELP]
>id="platform_privacyConsole_scheduleDatasetExpiration_instructions"
>title="手順"
>abstract="<ul><li>左側のナビゲーションで「<a href="https://experienceleague.adobe.com/docs/experience-platform/hygiene/ui/overview.html?lang=ja">データライフサイクル</a>」、「<b>リクエストを作成</b>」の順に選択します。</li><li>レコードを削除する場合：</li>   <li>「<b>レコード</b>」を選択します。</li>   <li>レコードを削除する特定のデータセットを選択するか、すべてのデータセットからレコードを削除するオプションを選択します。</li>   <li>レコードを削除する消費者の ID を指定します。「<b>ID を追加</b>」を選択して ID を 1 つずつ指定するか、「<b>ファイルを選択</b>」を選択して ID の JSON ファイルをアップロードします。</li>   <li>必要に応じて、「<b>テンプレート</b>」を選択して、JSON ファイルの予想される形式を表示します。</li><li><a href="https://experienceleague.adobe.com/docs/experience-platform/hygiene/ui/dataset-expiration.html?lang=ja#schedule-dataset-expiration">データセットの有効期限をスケジュール</a>する場合の手順については、ドキュメントを参照してください。</li></ul>"

リクエストを作成するには、ワークスペースのメインページから **[!UICONTROL 作成 リクエスト]** を選択します。

>[!IMPORTANT]
>
>Real-時間 CDP、Adobe Systems Journey Manager、および Customer Journey Analytics ユーザーには、保留中のスケジュール済みデータセット 有効期限作業指示書が 20 件あります。 ヘルスケア シールド、プライバシー、セキュリティ シールドのユーザーには、50 件の保留中のスケジュール済みデータセット 有効期限作業指示書があります。 つまり、一度に 20 個または 50 個のデータセットを削除するようにスケジュールできます。<br>たとえば、データセット の有効期限が 20 個予定されていて、明日 1 つのデータセットが削除される場合、そのデータセットが削除されるまで有効期限を設定することはできません。

![この [!UICONTROL データライフサイクル] workspace と [!UICONTROL リクエストを作成] ハイライト表示](../images/ui/ttl/create-request-button.png)

リクエスト作成ワークフローが表示されます。 の下 [!UICONTROL リクエストされたアクション] セクションで選択 **[!UICONTROL データセットを削除]** データセット有効期限スケジュール設定のコントロールを更新します。

![を使用したリクエスト作成ワークフロー [!UICONTROL データセットを削除] ハイライト表示されたオプション](../images/ui/ttl/dataset-selected.png)

### 日付およびデータセットの選択 {#select-date-and-dataset}

「**[!UICONTROL リクエストされたアクション]**」セクションで、データセットを削除する日付を選択します。日付を手動で入力するか(形式 `mm/dd/yyyy`内)、カレンダーアイコン(![カレンダーアイコン)を選択します。](../images/ui/ttl/calendar-icon.png))をクリックして、ダイアログから日付を選択します。

![データセットの有効期限日付が選択され、ハイライトされたカレンダーダイアログ。](../images/ui/ttl/select-date.png)

次へ、[データセット 詳細&#x200B;]**] でデータベース アイコン (**[!UICONTROL ![データベース アイコン。](../images/ui/ttl/database-icon.png))をクリックして、データセット選択ダイアログを開きます。 リストから有効期限を適用するデータセットを選択して、「**[!UICONTROL 完了]**」を選択します。

![この [!UICONTROL データセットを選択] データセットが選択され、 [!UICONTROL 完了] ハイライト表示](../images/ui/ttl/select-dataset.png)

>[!NOTE]
>
>現在のサンドボックスに属するデータセットのみが表示されます。

### リクエストの送信 {#submit-request}

「[!UICONTROL データセットの詳細]」セクションには、選択したデータセットのプライマリ ID とスキーマが表示されます。**[!UICONTROL リクエスト設定]**&#x200B;で、リクエストの名前とオプションの説明を入力し、「**[!UICONTROL 送信]**」を続行します。

![[!UICONTROL リクエスト設定]と[!UICONTROL 送信] ボタンがハイライトされた完了したデータセット 有効期限 リクエスト。](../images/ui/ttl/submit.png)

[!UICONTROL 確認 リクエスト]ダイアログが表示されます。データセット名とデータセットが削除される日付の確認が求められます。 「**[!UICONTROL 送信]**」をクリックして続行します。

リクエストが送信されると、作業指示が作成され、のメインタブに表示されます [!UICONTROL データライフサイクル] ワークスペース。 ここから、リクエストを処理する作業指示のステータスを監視できます。

>[!NOTE]
>
>データセットの有効期限が実行されるとどのように処理されるかの詳細については、[タイムラインと透明性](../home.md#dataset-expiration-transparency)の概要に関する節を参照してください。

## データセット有効期限の編集またはキャンセル {#edit-or-cancel}

データセットの有効期限を編集またはキャンセルするには、ワークスペースのメインページで&#x200B;**[!UICONTROL データセット]**&#x200B;を選択して、リストからデータセットの有効期限を選択します。

データセット有効期限の詳細ページで、右側のパネルにスケジュール設定された削除を編集またはキャンセルするためのコントロールが表示されます。
