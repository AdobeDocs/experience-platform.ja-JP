# 資格情報ガイド

Adobe Experience Platformクエリサービスを使用すると、外部クライアントと接続できます。 これらの外部クライアントに接続するには、期限が切れる資格情報か、期限が切れない資格情報を使用します。

## 資格情報の期限切れ

期限切れの資格情報を使用して、外部クライアントへの接続をすばやく設定できます。

![](../images/ui/credentials/expiring-credentials.png)

**[!UICONTROL 資格情報の期限切れ]** セクションには、次の情報が表示されます。

- **[!UICONTROL ホスト]**:接続先のホストの名前。クエリサービスに接続する場合は、現在使用している IMS 組織の名前が含まれます。
- **[!UICONTROL ポート]**:接続先のホストのポート番号。
- **[!UICONTROL データベース]**:接続先のデータベースの名前。
- **[!UICONTROL ユーザー名]**:クエリサービスへの接続に使用するユーザー名。
- **[!UICONTROL パスワード]**:クエリサービスへの接続に使用するパスワード。
- **[!UICONTROL PSQL コマンド]**:コマンドラインで PSQL を使用してクエリサービスに接続するためのすべての関連情報を自動的に挿入するコマンド。
- **[!UICONTROL 有効期限]**:期限が切れる資格情報の有効期限。資格情報は、生成されてから 24 時間で期限切れになります。

## 期限切れでない資格情報

期限切れでない資格情報を使用して、外部クライアントへのより恒久的な接続を設定できます。

### 前提条件

期限切れでない資格情報を生成する前に、Adobe Admin Consoleで次の手順を実行する必要があります。

1. [Adobe Admin Console](https://adminconsole.adobe.com/) にログインし、上部のナビゲーションバーから関連する組織を選択します。
2. [製品プロファイルの選択.](../../access-control/ui/browse.md)
3. [製品プロファイルのサン **** ドボックスと **クエリサービ** ](../../access-control/ui/permissions.md) ス統合の管理の権限の両方を設定します。
4. [新しいユーザーを製品プロファイルに追加](../../access-control/ui/users.md) し、ユーザーに設定済みの権限を付与します。
5. [ユーザーを製品プロファイル管理者として追加し](https://helpx.adobe.com/enterprise/using/manage-product-profiles.html) て、任意のアクティブな製品プロファイルのアカウント作成を許可します。
6. [統合を作成するには、ユーザーを製品プロ](https://helpx.adobe.com/enterprise/using/manage-developers.html) ファイル開発者として追加します。

権限の割り当て方法の詳細については、[ アクセス制御 ](../../access-control/home.md) のドキュメントを参照してください。

必要な権限がすべて、Adobe開発者コンソールで設定され、ユーザーが期限切れの資格情報機能を使用できるようになりました。

### 資格情報の生成

期限が切れない一連の資格情報を作成するには、Platform UI に戻り、左のナビゲーションから「**[!UICONTROL クエリ]**」を選択して、「[!UICONTROL  クエリ ]」ワークスペースにアクセスします。 次に、「**[!UICONTROL 資格情報]**」タブを選択し、「**[!UICONTROL 資格情報を生成]**」を選択します。

![](../images/ui/credentials/generate-credentials.png)

資格情報を生成するためのダイアログが表示されます。 期限が切れていない資格情報を作成するには、次の詳細を指定する必要があります。

- **[!UICONTROL 名前]**:生成する資格情報の名前。
- **[!UICONTROL 説明]**:（オプション）生成する資格情報の説明。
- ****&#x200B;に割り当て：資格情報が割り当てられるユーザー。この値は、資格情報を作成するユーザーの電子メールアドレスである必要があります。
- **[!UICONTROL パスワード]** （オプション）資格情報のオプションのパスワード。パスワードが設定されていない場合、Adobeは自動的にパスワードを生成します。

必要な詳細をすべて入力したら、「**[!UICONTROL 資格情報を生成]**」を選択して資格情報を生成します。

![](../images/ui/credentials/create-account.png)

>[!IMPORTANT]
>
>「**[!UICONTROL 資格情報を生成]**」ボタンを選択すると、設定 JSON ファイルがローカルマシンにダウンロードされます。 Adobeは生成された資格情報を **記録しない** ので、ダウンロードしたファイルを安全に保存し、資格情報の記録を保持する必要があります。
>
>また、資格情報が 90 日間使用されない場合は、資格情報が期限切れになります。

設定 JSON ファイルには、テクニカルアカウント名、テクニカルアカウント ID、資格情報などの情報が含まれます。 次の形式で指定します。

```json
{"technicalAccountName":"9F0A21EE-B8F3-4165-9871-846D3C8BC49E@TECHACCT.ADOBE.COM","credential":"3d184fa9e0b94f33a7781905c05203ee","technicalAccountId":"4F2611B8613AA3670A495E55"}
```

生成した資格情報を保存したら、「**[!UICONTROL 閉じる]**」を選択します。 期限が切れていないすべての資格情報のリストが表示されます。

![](../images/ui/credentials/list-credentials.png)

期限が切れていない資格情報は、編集または削除できます。 期限が切れていない秘密鍵証明書を編集するには、鉛筆アイコン (![](../images/ui/credentials/edit-icon.png)) を選択します。 期限が切れていない秘密鍵証明書を削除するには、削除アイコン (![](../images/ui/credentials/delete-icon.png)) を選択します。

期限が切れていない秘密鍵証明書を編集すると、モーダルが表示されます。 更新する次の詳細を指定できます。

- **[!UICONTROL 名前]**:生成する資格情報の名前。
- **[!UICONTROL 説明]**:（オプション）生成する資格情報の説明。
- ****&#x200B;に割り当て：資格情報が割り当てられるユーザー。この値は、資格情報を作成するユーザーの電子メールアドレスである必要があります。

![](../images/ui/credentials/update-credentials.png)

必要な詳細をすべて入力したら、「**[!UICONTROL アカウントを更新]**」を選択して、資格情報の更新を完了します。

## 資格情報を使用した外部クライアントへの接続

期限が切れる資格情報または期限が切れない資格情報を使用して、Aqua Data Studio、Looker、Power BIなどの外部クライアントと接続できます。 これらの資格情報の入力方法は、外部クライアントによって異なります。 これらの資格情報の使用に関する詳しい手順については、外部クライアントのドキュメントを参照してください。

この画像は、期限が切れていない資格情報のパスワードを除き、UI 内の各パラメーターの場所を示しています。 期限切れでない資格情報は JSON 設定ファイルで提供されますが、期限切れの資格情報は UI の「**資格情報**」タブに表示されます。

![](../images/ui/credentials/expiring-credentials.png)

次の表に、通常、外部クライアントへの接続に必要なパラメーターの概要を示します。

>[!NOTE]
>
>期限切れでない資格情報を使用してホストに接続する場合は、[!UICONTROL EXPIRING CREDENTIALS] セクションにリストされているすべてのパラメータを、パスワードとユーザー名を除いて使用する必要があります。

| パラメーター | 説明 |
|---|---|
| サーバー/ホスト | 接続先のサーバー/ホストの名前。 <ul><li>この値は、資格情報の期限が切れる場合と期限が切れない場合の両方に使用され、 `server.adobe.io` の形式を取ります。 この値は、[!UICONTROL EXPIRING CREDENTIALS] セクションの **[!UICONTROL Host]** にあります。</ul></li> |
| ポート | 接続先のサーバ/ホストのポート。 <ul><li>この値は、期限が切れる資格情報と期限が切れない資格情報の両方に使用され、 [!UICONTROL EXPIRING CREDENTIALS] セクションの **[!UICONTROL Port]** にあります。 ポートの値の例は `80` です。</ul></li> |
| データベース | 接続先のデータベース。 <ul><li>この値は、資格情報の期限が切れる場合と期限が切れない場合の両方に使用され、 [!UICONTROL EXPIRING CREDENTIALS] セクションの **[!UICONTROL Database]** にあります。 データベースの値の例は `prod:all` です。</ul></li> |
| ユーザー名 | 外部クライアントに接続するユーザーのユーザー名。 <ul><li>資格情報の期限が切れる前に使用する場合は、`@AdobeOrg` の前の英数字の文字列を使用します。 この値は **[!UICONTROL Username]** の下にあります。</li><li>期限が切れていない資格情報を使用している場合、これは選択した文字列ですが、**設定 JSON ファイルの `technicalAccountID` 値と同じにすることはできません**。</li></ul> |
| パスワード | 外部クライアントに接続するユーザーのパスワード。 <ul><li>資格情報の期限が切れる前に、[!UICONTROL EXPIRING CREDENTIALS] セクションの **[!UICONTROL Password]** に記載されています。</li><li>期限が切れない資格情報を使用している場合、この値は technicalAccountID からの引数と設定 JSON ファイルから取得した資格情報が連結された値になります。 パスワードの値は次の形式で指定します。`{technicalAccountId}:{credential}`.</li></ul> |

## 次の手順

これで、期限切れの資格情報と期限切れでない資格情報の両方の動作を理解できたので、これらの資格情報を使用して外部クライアントに接続できます。 外部クライアントの詳細については、『[ クライアントのクエリサービスへの接続ガイド ](../clients/overview.md)』を参照してください。
