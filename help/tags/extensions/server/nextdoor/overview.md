# [!DNL Nextdoor] Conversion API 拡張機能 – ユーザーガイド

## 概要

[!DNL Nextdoor] は、地域と人をつなぐソーシャルネットワーキングサービスです。 近所の人とコミュニケーションを取ったり、情報を共有したり、地元のイベントやニュースを最新の状態に保ったり、近所の人と商品を売買したりできるプラットフォームです。

[[!DNL Nextdoor] Conversion API Extension](https://help.nextdoor.com/s/article/About-the-Nextdoor-Conversion-API) を使用して、コンバージョンイベントを Conversion API に直接送信 [!DNL Nextdoor's] ます。 この拡張機能は、サーバーサイドのコンバージョンデータを送信することで、[!DNL Nextdoor] 広告キャンペーンのパフォーマンスを追跡および測定するのに役立ちます。

このガイドでは、イベント転送 [!DNL Nextdoor] ルール [&#x200B; で &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-platform/tags/ui/rules) Conversion API 拡張機能をインストール、設定、使用する方法について説明します。

## 前提条件 {#prerequisites}

この拡張機能を使用するには、有効な [!DNL Nextdoor] Ads Manager アカウントが必要です。 アカウントをまだお持ちでない場合は、[[!DNL Nextdoor Ads]  登録ページ &#x200B;](https://ads.nextdoor.com/v2/signup) に移動し、アカウントを登録して作成してください。

### 必要な設定の詳細の収集 {#configuration-details}

Experience Platformを [!DNL Nextdoor] に接続するには、以下が必要です。

| 資格情報 | 説明 | セキュリティ上の注意 |
| --- | --- | --- |
| データソース ID | [!DNL Nextdoor] からの一意のデータソース ID。Assets/ピクセルページにアクセスし、Nextdoor Pixel を生成することで、[!DNL Nextdoor Ads Manager] アカウントで見つけることができます。 | これを組織内で共有しても安全です。 |
| アクセストークン | 安全な通信のための API 認証アクセストークンです。 このトークンは、[!DNL Nextdoor Ads Manager] アカウントにログインして API 設定にアクセスすることで生成できます。 | このトークンはアカウントへのアクセスを提供するため、安全に保管してください。 |

## [!DNL Nextdoor] 拡張機能のインストールと設定 {#install}

拡張機能をインストールするには、左側のナビゲーションで「**[!UICONTROL Extensions]**」を選択します。 「**[!UICONTROL Catalog]**」タブで、**[!UICONTROL Nextdoor Conversion API Extension]** を選択し、「**[!UICONTROL Install]**」を選択します。

![&#x200B; インストールを強調表示した [!DNL Nextdoor] 拡張機能カードを示す拡張機能カタログ &#x200B;](../../../images/extensions/server/nextdoor/install-extension.png)

次の画面で、[!DNL Nextdoor Ads Manager] ークフローから生成した設定値を入力します。

* **[!UICONTROL Data Source ID]**
* **[!UICONTROL Access Token]**

終了したら「**[!UICONTROL Save]**」を選択します。

![[!DNL Nextdoor]&#x200B;[!DNL Nextdoor] 変換 API 拡張機能の設定画面 &#x200B;](../../../images/extensions/server/nextdoor/configure.png)

## イベント転送ルールの設定 {#config-rule}

すべてのデータ要素を設定したら、イベントを [!DNL Nextdoor] に送信するタイミングと方法を決定するイベント転送ルールを作成できます。

イベント転送プロパティに新しい [&#x200B; ルール &#x200B;](../../../ui/managing-resources/rules.md) を作成します。 **[!UICONTROL Actions]** の下で、新しいアクションを追加し、拡張機能を **[!UICONTROL Nextdoor Conversion API Extension]** に設定します。 Edge Network イベントを [!DNL Nextdoor] に送信するには、**[!UICONTROL Action Type]** を **[!UICONTROL Report Web Conversions]** に設定します。

![&#x200B; データ収集 UI で [!UICONTROL Report Web Conversions] ルールに対して選択されている [!DNL Nextdoor] アクションタイプ &#x200B;](../../../images/extensions/server/nextdoor/select-action.png)

この選択を行うと、以下に示すように、イベントをさらに設定できる追加のコントロールが表示されます。 完了したら、「**[!UICONTROL Keep Changes]**」を選択してルールを保存します。

* **既存のデータ要素を選択**：既に作成されているデータ要素から選択します。

* **動的な値を適用**:Web サイトをソースとする動的コンテンツをフィールドに入力します。

## ベストプラクティス {#best-practices}

正確なコンバージョントラッキングの確保、データ品質の向上、プライバシー規制の遵守については、次のベストプラクティスに従います。 これらの推奨事項は、[!DNL Nextdoor] コンバージョン API 統合の有効性を最大化し、広告パフォーマンスを最適化するのに役立ちます。

### データ品質とプライバシーコンプライアンス

ユーザーのプライバシーと規制への準拠を維持しながら、コンバージョン属性の精度を最大限に高めるには、適切なデータ処理が不可欠です。 これらのプラクティスは、顧客データを正しくフォーマットし、安全に処理し、プライバシー規制に従って処理するのに役立ちます。

* **一貫したデータ形式の使用**：メール、電話番号、その他のデータの形式を一貫して設定する：

   * **メールの正規化**：メールを小文字に変換し、空白をトリミングし、[!DNL Gmail] アドレスからドットを削除します。
   * **電話番号の標準化**：国際互換性には E.164 形式（+1234567890）を使用します。
   * **アドレスの正規化**：番地の略語（St.、Ave.、Rd.）を標準化し、余分なスペースを削除します。
   * **名前の書式設定**：名前を小文字に変換し、余分なスペースを削除し、特殊文字を一貫して処理します。

* **機密データのハッシュ化**：機密の顧客データを送信する前にハッシュ化することを検討してください。 一貫性のあるハッシュ値を確保するために、ハッシュ化の前にデータを常に正規化します。

   * 電話番号、アドレス、その他の PII には SHA-256 ハッシュを使用します。
   * 拡張機能では、プレーンテキストのメールが自動的にハッシュ化されます。どちらの形式でも送信できます。
   * すべてのトラッキング実装全体でデータを一貫してハッシュ化します。
      * **例**:「John@Example.COM」を「john@example.com」→正規化してからハッシュ化します。

* **完全な情報を提供**：マッチングを向上させるために、できるだけ多くの顧客情報を含めます。

   * 使用可能な場合は、複数の識別子（メール +電話またはメール +名前+ アドレス）を含めます。
   * 外部顧客 ID を使用すると、コンバージョンイベントを CRM データにリンクできます。
   * 可能な場合は人口統計データ（年齢、性別）を提供し、プライバシー法に準拠します。

* **同意管理**：データ収集に対する適切な同意があることを確認します。

   * 顧客データを収集する前に、適切な同意メカニズムを実装する。
   * ユーザーのオプトアウト環境設定とデータ削除リクエストに従います。
   * オプトアウトしたユーザーには、制限付きのデータ使用パラメーターを使用します。
   * **リソース**:
      * [GDPR コンプライアンスガイド &#x200B;](https://gdpr.eu/compliance/)
      * [CCPA プライバシー要件 &#x200B;](https://oag.ca.gov/privacy/ccpa)

* **データの最小化**：コンバージョントラッキングに必要なデータのみを送信します。

   * アトリビューションと最適化に不可欠なデータのみを収集して送信します。
   * 送信するデータフィールドを定期的に確認し、監査します。
   * 不要な顧客情報を削除して、プライバシーリスクを軽減します。

* **正確なタイムスタンプの使用**：適切なアトリビューションに対して、イベントのタイムスタンプが正確であることを確認します。
   * データを処理した時刻ではなく、コンバージョンが発生した実際の時刻を使用します。
   * タイムスタンプが Unix エポック形式（1970 年 1 月 1 日（PT）以降の秒数）であることを確認します。
   * タイムスタンプ計算でのタイムゾーンの違いを考慮します。

### イベントの重複排除

キャンペーンの正確な測定を確保し、パフォーマンス指標の水増しを避けるために、コンバージョンイベントの重複を防ぎます。 適切な重複排除を実装して、各コンバージョンが 1 回だけカウントされるようにし、最適化の決定に信頼性の高いデータを提供します。

* **一意のイベント ID を指定**：常に一意のイベント ID を使用して、コンバージョンの重複を防ぎます。
* **一貫した命名を使用**：実装全体で一貫したイベント命名を適用します。

### レート制限

[!DNL Nextdoor] Conversion API には、レート制限が適用されます。 現在のレート制限は、広告主ごとに **100 リクエスト/分** です。 レート制限を超えた場合、API は `HTTP 429 Too Many Requests` エラーコードを返します。
