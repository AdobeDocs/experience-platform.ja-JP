# ビルドエンドポイント

拡張機能、ルールおよびデータ要素は、Adobe Experience Platform のタグの構築ブロックです。 アプリケーションで何らかの処理を実行すると、これらの構築ブロックは[ライブラリ](./libraries.md)に追加されます。エクスペリエンスアプリケーションにライブラリをデプロイするために、ライブラリはビルドにコンパイルされます。 Reactor API の `/builds` エンドポイントを使用すると、エクスペリエンスアプリケーション内のビルドをプログラムで管理できます。

ビルドとは、Web およびモバイルアプリケーション内で読み込まれる実際のファイルのことです。各ビルドの内容は、次の要因によって異なります。

* ライブラリに含まれるリソース
* ライブラリが構築される[環境](./environments.md)の設定
* ビルドが属する[プロパティ](./properties.md)のプラットフォーム

ビルドは、1 つのライブラリのみに属します。ライブラリには多くのビルドを含めることができます。

ビルドの一般的な情報と、ビルドがタグの公開ワークフローにどのように適合するかについては、[公開の概要](../../ui/publishing/overview.md)を参照してください。

## はじめに

このガイドで使用するエンドポイントは、[Reactor API](https://www.adobe.io/experience-platform-apis/references/reactor/) の一部です。続行する前に、 [はじめる前に](../getting-started.md) で、API への認証方法に関する重要な情報を確認してください。

## ビルドのリストの取得 {#list}

GET リクエストのパスにライブラリの ID を含めることで、特定のライブラリのビルドをリスト表示できます。

**API 形式**

```http
GET /libraries/{LIBRARY_ID}/builds
```

| パラメーター | 説明 |
| --- | --- |
| `LIBRARY_ID` | ビルドのリストを取得するライブラリの `id`。 |

{style=&quot;table-layout:auto&quot;}

>[!NOTE]
>
>クエリパラメーターを使用して、リストされたビルドを次の属性に基づいてフィルタリングできます。<ul><li>`created_at`</li><li>`status`</li><li>`token`</li><li>`updated_at`</li></ul>詳しくは、 [応答のフィルタリング](../guides/filtering.md) に関するガイドを参照してください。

**リクエスト**

```shell
curl -X GET \
  https://reactor.adobe.io/libraries/LBad32d71feff844b7b5a11dd0bf030964/builds \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {IMS_ORG}' \
  -H "Content-Type: application/vnd.api+json" \
  -H 'Accept: application/vnd.api+json;revision=1'
```

**応答**

応答が成功すると、指定されたライブラリのビルドのリストが返されます。

## ビルドの作成 {#create}

POST リクエストのパスにライブラリの ID を含むライブラリのビルドを作成できます。

**API 形式**

```http

```

| パラメーター | 説明 |
| --- | --- |
| `LIBRARY_ID` | ビルドを定義するライブラリの `id`。 |

{style=&quot;table-layout:auto&quot;}

**リクエスト**

次のリクエストは、リクエストパスで指定されたライブラリの新しいビルドを作成します。 リクエストペイロードは不要です。

```shell

  -H 'x-api-key: {API_KEY}' \
```

## ビルドの再公開 {#republish}

PATCH リクエストのパスに ID を含めることで、[公開ライブラリ](./libraries.md#publish)からビルドを再公開できます。

**API 形式**

```http
PATCH /builds/{BUILD_ID}
```

| パラメーター | 説明 |
| --- | --- |
| `BUILD_ID` | 再公開するビルドの `id`。 |

{style=&quot;table-layout:auto&quot;}

**リクエスト**

次のリクエストは、既存のアプリ設定の `app_id` を更新します。

```shell
curl -X PATCH \
  https://reactor.adobe.io/builds/BLb408c04c20ba4a82b6df496969a99781 \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {IMS_ORG}' \
  -H 'Content-Type: application/json' \
  -d '{
        "data": {
          "id": "BLb408c04c20ba4a82b6df496969a99781",
          "type": "builds",
          "meta": {
            "action": "republish"
          }
        }
      }'
```

| プロパティ | 説明 |
| --- | --- |
| `id` | 更新するビルドの `id`。この値は、リクエストパスで指定された `{BUILD_ID}` 値と一致する必要があります。 |
| `type` | 更新するリソースのタイプ。 このエンドポイントの場合は、値を `builds` にする必要があります。 |
| `meta.action` | 実行する PATCH アクションのタイプ。`republish` に設定する必要があります。 |

{style=&quot;table-layout:auto&quot;}

**応答**

応答が成功すると、再公開されたビルドの詳細が返されます。

## ビルドの関連リソースの取得 {#related}

```shell
curl -X GET \
```
