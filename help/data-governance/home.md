# データガバナンスの概要

Adobe Experience Platformの主な機能の1つは、複数のエンタープライズシステムからのデータを統合して、マーケティング担当者が顧客をより良く識別、理解および関与できるようにすることです。 このデータは、組織または法規制によって定義された使用制限の対象となる場合があります。したがって、プラットフォーム内のデータ操作がデータ使用ポリシーに準拠していることを確認することが重要です。

Adobe Experience Platform データガバナンスを使用すると、顧客データを管理し、データの使用に適した規制、制限、ポリシーへのコンプライアンスを確保できます。Experience Platformでは、様々なレベルで重要な役割を果たします。例えば、カタログ、データ系列、データ使用状況のラベル付け、データ使用ポリシー、マーケティングアクションに対するデータの使用制御などです。

## データ管理の役割

概念として、データガバナンスは自動的でも、真空でも発生しません。 1人の個人の役割として始まったものは、 **データ管理者として一般的に認められ**、データ・ガバナンスのエコシステムが拡大するにつれて、大きく成長しています。 今日、データ・ガバナンスを成功させるためには継続的な管理と監視が必要で、データに適切なラベル付けが可能なツール、使用ポリシーの作成、ポリシーの遵守を実施できるデータ管理者に依存しています。

データ・ガバナンスは組織内の各個人の責任である必要がありますが、データ・ガバナンス・サイクルにおける重要な役割は次のとおりです。

![データ管理の役割](./images/overview/roles.png)

### データ管理

データ管理は、データ管理の中心です。 この役割は、規則、契約上の制限、ポリシーを解釈し、データに直接適用する責任を負います。 データ管理者は、これらの規則、制限、ポリシーを理解することで、次のような役割を果たします。

* メタデータの使用に関するラベル付けを適用および管理するための、データ、データセット、データサンプルの確認。
* データポリシーを作成し、データセットとフィールドに適用する。
* 組織へのデータポリシーの通信。

### マーケター

マーケティング担当者は、データ管理のエンドポイントです。 データ管理者、科学者、エンジニアが作成したデータ管理インフラストラクチャからデータを要求します。 マーケティング担当者は、次のような、様々なマーケティングのかさの下の専門分野を取り上げます。

* マーケティングアナリストは、顧客を個人として、またはグループ（セグメントとも呼ばれる）の両方に理解できるようにデータを要求します。
* マーケティングスペシャリストおよびエクスペリエンスデザイナーは、データを使用して新しい顧客エクスペリエンスを設計します。


## DULEフレームワーク

Data Usage Labeling and Enforcement(DULE)は、エクスペリエンスプラットフォームデータガバナンスの中核的なフレームワークです。 SCHEDULEを使用すると、データの分類とデータ使用ポリシーの作成のプロセスが簡素化および合理化されます。 データラベルが適用され、データ使用ポリシーが設定されると、マーケティングアクションを評価して、データが正しく使用されていることを確認できます。

DULEフレームワークには、3つの主要な要素があります。 ラベル、ポリシー、実施。

1. **ラベル：** プライバシーに関する考慮事項と契約条件を反映したデータを分類し、規制や組織のポリシーに準拠するようにします。
1. **ポリシー：** 特定のデータに対して実行を許可する、または実行を許可しないマーケティングアクションの種類を説明します。
1. **実施：** ポリシーフレームワークを使用して、様々なデータアクセスパターンにわたってポリシーに対するアドバイスと適用を行います。

## データ使用量ラベル

Data Governanceを使用すると、データステッダーは、使用状況ラベルをデータセットとフィールドレベルで適用し、適用されるポリシーのタイプに従ってデータを分類できます。

DULEフレームワークには、次の4つの方法でデータを分類するために使用できる事前定義のデータ使用ラベルが含まれています。

![データ使用ラベルのカテゴリ](./images/overview/label-categories.png)

* **契約「C」のデータラベル：** 契約上の義務を負うデータ、または顧客のデータ管理ポリシーに関連するデータにラベルを付け、分類します。
* **ID 「I」データラベル：** 特定の人物を識別または連絡できるデータにラベルを付け、分類します。
* **機密性の高い「S」データラベル：** 地理的データなどの機密データに関連するデータにラベルを付け、分類する。
* **GDPRデータ・ラベル：** GDPRアクセス/削除要求で使用する個人IDを含む可能性のあるデータにラベルを付け、分類します。

>[!NOTE] 使用可能なラベルの完全なリスト、および各ラベルタイプの定義については、 [サポートされるデータ使用ラベルのガイドを参照してください](labels/reference.md) 。

ラベルはいつでも適用でき、データの管理方法を柔軟に選択できます。 ベストプラクティスでは、データがエクスペリエンスプラットフォームに取り込まれるとすぐに、またはデータがプラットフォームで使用可能になるとすぐに、データにラベル付けを行うことを推奨します。

UIを使用してデータセットとフィールドにラベルを適用する方法については、 [データ使用ラベル](./labels/overview.md) (DATA USAGE LABEL)の概要を参照してください。

## データ使用ポリシー

データ使用ラベルがデータのコンプライアンスを効果的にサポートするためには、データ使用ポリシーを実装する必要があります。 データ使用ポリシーとは、エクスペリエンスプラットフォーム内のデータに対して実行を許可（制限）されるマーケティングアクションの種類を記述するルールです。

マーケティングアクションの例としては、データセットをサードパーティのサービスにエクスポートしたい場合などが考えられます。 個人識別情報(PII)などの特定の種類のデータをエクスポートできず、「I」ラベル（IDデータ）がデータセットに適用されているというポリシーが設定されている場合、Policy Serviceからデータ使用ポリシーに違反したことを伝える応答が返されます。

データ使用ラベルが適用されると、データ管理者はDULE Policy Service APIまたはExperience Platformユーザーインターフェイスを使用してポリシーを作成できます。

DULE Policy Service APIが提供する主要な操作の実行について詳しくは、 [Policy Service開発者ガイドを参照してください](api/getting-started.md)。 DULEポリシーを使用する手順については、APIを使用したDULEポリシーの [作成と評価に関するチュートリアルを参照してください](policies/create.md)。

Experience Platform UIでポリシーを管理する方法について詳しくは、 [ポリシーユーザーガイドを参照してください](policies/user-guide.md)。

## 今後のリリース

Data Governanceは、現在、2つのレベル（データセットとフィールド）でDULEラベル付けをサポートしています。 Data Governanceは、DULE Policy Service APIを使用したデータ使用ポリシーおよびマーケティングアクションの作成と管理もサポートします。

以降のリリースでは、次の機能が提供されます。

* カスタムデータ使用ラベル： 組織のニーズに基づいて新しいラベルと定義を作成します。
* ポリシーの適用： ポリシーフレームワークを使用して、様々なデータアクセスパターンにわたってポリシーを通知し、適用します。
* 監査： データ・アクセス・アクティビティを監視し、コンプライアンスの問題を特定してレポートします。

次の節では、データガバナンスに関する追加情報を示します。

### データ・ガバナンスの用語

次の表に、Data GovernanceとDULEフレームワークに関連する主な用語の概要を示します。

| 用語 | 定義 |
|---|---|
| **契約ラベル** | 契約「C」ラベルは、契約上の義務を負うデータ、または組織のデータ管理ポリシーに関連するデータの分類に使用されます。 |
| **クロスサイトデータ** | クロスサイトデータは、オンサイトデータとオフサイトデータの組み合わせ、または複数のオフサイトソースのデータの組み合わせを含む、複数のサイトのデータの組み合わせです。 |
| **データ管理** | データ・ガバナンスには、データの使用に関する規制や企業ポリシーに準拠したデータを確実に保証するために使用される戦略とテクノロジーが含まれます。 |
| **データ管理** | データ管理者は、組織のデータアセットの管理、監視、および実施を担当する人です。 また、データ管理担当者は、政府の規制や組織のポリシーに準拠するように、データ管理ポリシーを保護し、保守することもできます。 |
| **データ使用量ラベル** | データ使用ラベルを使用すると、プライバシーに関する考慮事項や契約条件を反映したデータを、規制や企業ポリシーに準拠するように分類できます。 |
| **データセットのラベル** | ラベルをデータセットに追加できます。 データセット内のすべてのフィールドは、データセットのラベルを継承します。 |
| **DULE** | DULEは、「Data Usage Labeling and Enforcement」の略語です。 データ・ガバナンスの重要な部分であるDULEは、データ使用のラベル付けを可能にし、組織内のガバナンス・ニーズに合わせてデータ・アクセス・ポリシーを適用する機能の集まりです。 |
| **フィールドラベル** | フィールドラベルは、データセットから継承される、またはフィールドに直接適用されるデータガバナンスラベルです。  フィールドに適用したデータ管理ラベルは、データセットに継承されません。 |
| **ジオフェンス** | ジオフェンスとは、GPSまたはRFIDテクノロジーで定義される仮想的な地理的境界です。モバイルデバイスが特定の領域に入るか離れるときにソフトウェアが応答をトリガーするようにします。 |
| **IDラベル** | ID「I」ラベルは、特定の人物を識別または連絡できるデータを分類するために使用します。 |
| **関心に基づくターゲティング** | 関心ベースのターゲティング（パーソナライゼーションとも呼ばれます）は、次の3つの条件が満たされた場合に発生します。 オンサイトで収集されたデータは、ユーザーの興味を引くために使用され、別のサイトやアプリ（オフサイト）などの別のコンテキストで使用され、それらの推定に基づいて提供されるコンテンツや広告を選択するために使用されます。 |
| **マーケティングのアクション** | マーケティングアクションは、データ管理フレームワークの観点から見ると、Experience Platformのデータコンシューマーが行うアクションで、データ使用ポリシーの違反を確認する必要があります |
| **ポリシー** | データ管理フレームワークでは、ポリシーとは、特定のデータに対して実行を許可する、または実行を許可しないマーケティングアクションの種類を説明するルールです。 |
| **機密ラベル** | 機密性の高い「S」ラベルは、自分が属するデータを分類する際に使用され、組織では機密性の高いデータを考慮します。 |
