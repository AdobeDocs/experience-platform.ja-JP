# サポートされるデータ使用ラベル

Adobe Experience Platformには、データ使用量のラベル付けと実施(DULE)を中核とするデータガバナンスのインフラストラクチャが含まれています。  DULE機能を使用すると、データに適用される使用ポリシーのタイプに従ってデータを分類するために、データ使用ラベルをデータセットやフィールドに適用できます。

次のリストでは、Experience Platformで現在サポートされているすべてのデータ使用ラベルの概要を説明します。

データガバナンスとSDULEの詳細については、「データガバナンスの概要」を [参照してください](../home.md)。

## 契約ラベル

契約「C」のラベルは、契約上の義務を負うデータや、組織のデータガバナンスポリシーに関連するデータを分類するために使用されます。

| ラベル | 定義 |
|---|---|
| **C1** | データは、個々の識別子やデバイスの識別子を含めずに、集計されたフォームでのみAdobe Experience Cloudからエクスポートできます。 [詳細情報...](#c1) |
| **C2** | データをサードパーティにエクスポートすることはできません。 [詳細情報...](#c2) |
| **C3** | データを組み合わせたり、直接識別可能な情報と組み合わせて使用したりすることはできません。 [詳細情報...](#c3) |
| **C4** | データは、オンサイトまたはクロスサイトのどの広告やコンテンツのターゲット設定にも使用できません。 [詳細情報...](#c4) |
| **C5** | データは、コンテンツや広告の関心に基づく、サイト間のターゲティングには使用できません。 [詳細情報...](#c5) |
| **C6** | データは、オンサイト広告のターゲット設定には使用できません。 [詳細情報...](#c6) |
| **C7** | データは、コンテンツのオンサイトターゲティングには使用できません。 [詳細情報...](#c7) |
| **C8** | 組織のWebサイトやアプリの測定にデータを使用することはできません。 [詳細情報...](#c8) |
| **C9** | データは、データサイエンスワークフローでは使用できません。 [詳細情報...](#c9) |

## IDラベル

ID「I」ラベルは、特定の人物を識別または連絡できるデータを分類するために使用されます。

| ラベル | 定義 |
|---|---|
| **I1** | デバイスではなく、特定の人物を識別したり、特定の人に連絡したりできる、直接識別可能なデータ。 |
| **I2** | 間接的に識別可能なデータ。他のデータと組み合わせて、特定の人物を識別または連絡するために使用できます。 |

## 機密ラベル

機密性の高い「S」ラベルは、自分と組織で機密性の高いデータを分類するために使用されます。

機密性が高いと考えられるデータの1つは、異なるタイプの地理データである場合があります。ただし、このカテゴリは地理的データに限られません。

| ラベル | 定義 |
|---|---|
| **S1** | デバイスの正確な位置を判断するために使用できる緯度と経度を指定するデータ。 |
| **S2** | 広く定義されたジオフェンス領域を決定するために使用できるデータ。 |


## 詳細情報

次の節では、特定のラベルの実装に関する詳細情報を示します。

### C1 {#c1}

一部のデータは、個々の識別子やデバイスの識別子を含めずに、集計されたフォームでのみAdobe Experience Cloudからエクスポートできます。 例えば、ソーシャルネットワークから生成されたデータなどです。

### C2 {#c2}

一部のデータプロバイダーは、契約の条件で、最初に収集された場所からのデータのエクスポートを禁止しています。 例えば、ソーシャルネットワークの契約によって、多くの場合、ソーシャルネットワークから受け取るデータの転送が制限されます。 C2ラベルは [C1よりも制限が厳しいので](#c1)、集計と匿名データのみが必要です。

### C3 {#c3}

一部のデータプロバイダーでは、契約に条件が含まれており、そのデータと直接識別可能な情報の組み合わせや使用を禁止しています。 例えば、広告ネットワーク、広告サーバー、およびサードパーティのデータプロバイダーをソースとするデータの契約には、直接識別可能なデータを含むデータの使用に関する特定の契約上の禁止が含まれる場合が多くあります。

### C4 {#c4}

C4は最も厳しいラベルで、 [C5](#c5)、 [C6](#c6)、 [C7](#c7)を含みます。

### C5 {#c5}

次の3つの条件が満たされた場合、関心ベースのターゲティング（パーソナライゼーション）が発生します。オンサイトで収集されたデータは、(1)ユーザの興味に関する推論に使用され、(2)は別のサイトやアプリ（オフサイト）などの別のコンテキストで使用され、(3)は、それらの推測に基づいて提供されるコンテンツや広告を選択します。

複数のサイトのデータの組み合わせ（オンサイトデータとオフサイトデータの組み合わせ、または複数のオフサイトソースのデータの組み合わせなど）は、クロスサイトデータと呼ばれます。 異なるサイトは異なるコンテキストを表し、クロスサイトデータのコンテキストでの使用が元のデータと異なるようになります。 通常、クロスサイトデータは収集され、処理されて、ユーザの関心事に関する推論が行われます。 その結果、広告やコンテンツをターゲットにするためのクロスサイトデータの使用は、通常、広告やコンテンツがオンサイトに表示されるかオフサイトに表示されるかに関係なく、関心に基づくターゲティングの対象となります。 例えば、オンサイトデータをオフサイトデータと組み合わせて使用し、組織のサイトでユーザーを表示する広告を選択した場合、その使用は関心に基づくターゲティングの対象となります。 別の例として、オフサイトのユーザーに対する広告の再ターゲティングも、興味に基づくターゲティングと見なされる可能性が高くなります。

オフサイトデータは通常、ユーザの興味に関する推論を行うために収集され、処理されるので、ターゲティング用のオフサイトデータのみを使用する場合も、関心ベースのターゲティングと見なされます。

ただし、オンサイトデータのみを使用してコンテンツや広告をターゲット設定する場合、通常は関心に基づくターゲット設定とは見なされません。 他の方法では関心に基づくターゲティングと見なされないオンサイトターゲティングは、2つの異なるラベルとして扱われます。 特に、ラベルC6はオンサイト広告のターゲット設定とレポートに対応し、特に広告の選択、配信、レポートに対応し、ラベルC7はオンサイトコンテンツの選択、配信、レポート（オンサイトコンテンツのターゲット設定）に対応します。

最終的に、ラベルの解釈と、そのラベルを使用したデータの使用の強制方法は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB:パーソナライゼーション。 他のWebサイトやアプリなどの他のコンテキストで、時間の経過と共に広告やコンテンツをパーソナライズするために、このサービスの使用に関する情報の収集と処理。 通常、サイトやアプリのコンテンツは、広告やコンテンツの将来の選択を知らせる関心事に関する推論を行うために使用されます。

DAA:オンライン行動広告。 特定のコンピュータまたはデバイスから、時間の経過と非アフィリエイトWebサイト間でのWeb表示行動に関するデータを収集し、ユーザの好みや関心を予測し、Web表示行動から推測される好みや興味に基づいてそのコンピュータまたはデバイスに広告を配信する。

### C6 {#c6}

広告とは、主に商品やサービスの販売を促進する目的でWebサイトやアプリに表示される、テキストや画像を含むメッセージや通知のことです。 このようなメッセージや通知の目的は、ユーザーが決定します。 広告は、オンサイトコンテンツとは別のもので、ラベル [C7でカバーされます](#c7)。 C6ラベルの付いたデータは、組織のWebサイトやアプリに対する広告の選択と配信、またはそのような広告の配信と効果の測定など、オンサイト広告のターゲット設定には使用できません。 これには、以前に収集したユーザの興味に関するオンサイトデータを使用して、広告を選択し、何の広告が表示されたか、いつどこで表示されたか、広告のクリックや購入など、広告に関する何らかのアクションをユーザが行ったかが含まれます。 一般に、ユーザーのオンサイトアクティビティに基づいてユーザーの環境設定を参照し、オンサイト広告ターゲットでこれらの環境設定を使用しても、関心に基づくターゲット設定に必要な3つの要件をすべて満たさないので、関心に基づくターゲット設定（パーソナライゼーション）にはなりません。 _[これらの要件については、ラベルC5を参照してください。](#c5)_

最終的に、ラベルの解釈と、そのラベルを使用したデータの使用の強制方法は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB:3. 広告選択，配信,レポート:情報の収集と、以前に収集した情報との組み合わせ、広告を選択して配信し、そのような広告の配信と効果を測定する。 これには、以前に収集した関心に関する情報を使用して広告を選択し、表示された広告に関するデータを処理し、表示された頻度、日時と場所、広告のクリックや購入など、広告に関するアクションを実行したかどうかが含まれます。 これには、パーソナライゼーションは含まれません。パーソナライゼーションとは、このサービスの使用に関する情報を収集し、処理して、Webサイトやアプリなどの他のコンテキストでの広告やコンテンツのパーソナライズを後で行うことです。

DAA:オンライン行動分析には、ファーストパーティ、広告配信、広告レポート、コンテキスト広告(訪問中のWebページのコンテンツに基づく広告、Webページへの消費者の現在の訪問、検索クエリ)のアクティビティは含まれません。

### C7 {#c7}

オンサイトコンテンツとは、通知、教育、接待を目的として設計され、商品やサービスの販売を促進する目的で作成されないテキストや画像のことです。 コンテンツがネイティブの広告と見なされるかどうかなど、コンテンツの目的はユーザーが決定します。 C7ラベルは、オンサイト広告をカバーするものではなく、ラベル [C6でカバーされます](#c6)。 C7ラベルの付いたデータは、組織のWebサイトやアプリでのコンテンツの選択や配信、またはそのようなコンテンツの配信と効果の測定など、オンサイトコンテンツのターゲット設定には使用できません。 これには、選択したコンテンツに対するユーザの興味、表示されたコンテンツに関するデータの処理、表示されたコンテンツの頻度や表示時間、表示された日時や場所、コンテンツのクリックなど、コンテンツに関するアクションを実行したかどうかに関する情報が含まれます。 一般に、ユーザーのオンサイトアクティビティに基づいてユーザーの環境設定を参照し、その環境設定をオンサイトコンテンツのターゲット設定で使用しても、関心に基づくターゲット設定に必要な3つの要件をすべて満たさないので、関心に基づくターゲット設定は行われません。 _[これらの要件については、ラベルC5を参照してください。](#c5)_

最終的に、ラベルの解釈と、そのラベルを使用したデータの使用の強制方法は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB:4. コンテンツ選択、配信、レポート:情報の収集、および以前に収集した情報との組み合わせにより、コンテンツを選択して配信し、そのコンテンツの配信と効果を測定します。 これには、以前に収集した興味に関する情報を使用して、コンテンツを選択し、コンテンツが表示された回数や表示時間に関するデータを処理し、コンテンツが表示された日時や場所、コンテンツのクリックなど、コンテンツに関するアクションを実行したかどうかが含まれます。 これには、パーソナライゼーションは含まれません。パーソナライゼーションとは、Webサイトやアプリなど他のコンテキストでのコンテンツや広告のパーソナライズを、このサービスの使用に関する情報の収集と処理です。

DAA:オンライン行動分析には、ファーストパーティ、広告配信、広告レポート、コンテキスト広告(訪問中のウェブページの内容に基づく広告、Webページへの消費者の現在の訪問、検索クエリ)のアクティビティは含まれません。

### C8 {#c8}

データを使用して、組織のサイトやアプリのユーザーの使用状況を測定、把握、レポートすることはできません。 これには、興味に基づくターゲティング（クロスサイトターゲティング）は含まれません。これは、Webサイトやアプリなど他のコンテキストでコンテンツや広告を長期的にパーソナライズするために、このサービスを使用する方法に関する情報を集めたものです。

### C9 {#c9}

一部の契約には、データ科学のためのデータ使用に対する明示的な禁止が含まれます。 Artificial Intelligence(AI)、機械学習(ML)、モデリングでのデータの使用を禁止する用語で表現される場合があります。
