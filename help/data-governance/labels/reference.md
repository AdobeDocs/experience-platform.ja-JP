# サポートされるデータ使用量ラベル

Adobe Experience Platformには、Data Usage Labeling and Enforcement(DULE)を中核とするデータ・ガバナンスのインフラストラクチャが含まれます。  DULE機能を使用すると、データ使用ラベルをデータセットやフィールドに適用し、そのデータに適用される使用ポリシーの種類に従ってデータを分類できます。

このドキュメントでは、Experience Platformが現在サポートしているすべてのデータ使用ラベルについて概説します。 Data GovernanceとDULEに関する詳細は、「 [Data Governance overview](../home.md)」を参照してください。

## 契約ラベル

契約「C」ラベルは、契約上の義務を負うデータ、または組織のデータ管理ポリシーに関連するデータの分類に使用されます。

| ラベル | 定義 |
|---|---|
| **C1** | データは、個々の識別子やデバイスの識別子を含めずに、集計されたフォームでのみAdobe Experience Cloudからエクスポートできます。 [詳細情報...](#c1) |
| **C2** | データをサードパーティにエクスポートすることはできません。 [詳細情報...](#c2) |
| **C3** | データを組み合わせたり、直接識別できる情報と組み合わせて使用したりすることはできません。 [詳細情報...](#c3) |
| **C4** | データは、オンサイトまたはクロスサイトのどの広告やコンテンツのターゲット設定にも使用できません。 [詳細情報...](#c4) |
| **C5** | データは、コンテンツや広告の関心に基づくクロスサイトターゲティングには使用できません。 [詳細情報...](#c5) |
| **C6** | データは、オンサイト広告のターゲット設定には使用できません。 [詳細情報...](#c6) |
| **C7** | データは、コンテンツのオンサイトターゲティングには使用できません。 [詳細情報...](#c7) |
| **C8** | データを組織のWebサイトやアプリの測定に使用することはできません。 [詳細情報...](#c8) |
| **C9** | データは、Data Scienceワークフローでは使用できません。 [詳細情報...](#c9) |
| **C10** | データはステッチIDアクティベーションには使用できません。 [詳細情報...](#c10) |

## IDラベル

ID「I」ラベルは、特定の人物を識別または連絡できるデータを分類するために使用します。

| ラベル | 定義 |
|---|---|
| **I1** | デバイスではなく、特定の人物を識別したり、連絡先を特定したりできる、直接識別可能なデータ。 |
| **I2** | 間接的に識別可能なデータ。他のデータと組み合わせて、特定の人物を特定したり、特定の人物に連絡したりするのに使用できます。 |

## 機密ラベル

機密性の高い「S」ラベルは、自分が属するデータを分類する際に使用され、組織では機密性の高いデータを考慮します。

機密性が高いと考えられるデータの1つのタイプは、異なるタイプの地理的データである場合があります。 ただし、このカテゴリは地理的なデータに限られません。

| ラベル | 定義 |
|---|---|
| **S1** | デバイスの正確な位置を判断するのに使用できる緯度と経度を指定するデータ。 |
| **S2** | 広く定義されたジオフェンス領域を決定するのに使用できるデータ。 |

## 付録

以下の節では、使用可能なデータ使用ラベルについて詳しく説明します。

### 契約ラベルの詳細

以下の各項では、特定の契約Cのラベルの実装に関する詳細情報について説明します。

#### C1 {#c1}

一部のデータは、個々の識別子やデバイスの識別子を含めずに、集計したフォームでAdobe Experience Cloudからのみエクスポートできます。 例えば、ソーシャルネットワークから派生したデータなどです。

#### C2 {#c2}

一部のデータプロバイダは、契約に含まれる条件に基づき、最初に収集されたデータのエクスポートを禁止しています。 例えば、ソーシャルネットワークの契約によって、ユーザーがソーシャルネットワークから受け取るデータの転送が制限される場合があります。 C2ラベルは [C1](#c1)よりも制限が厳しく、集計と匿名データのみが必要です。

#### C3 {#c3}

一部のデータプロバイダーは契約に条件を持っており、そのデータと直接識別できる情報との組み合わせや使用を禁止しています。 例えば、広告ネットワーク、広告サーバー、サードパーティのデータプロバイダーなどのデータソースの契約では、データを直接識別できる形で使用する場合、特定の契約上の禁止が含まれます。

#### C4 {#c4}

C4は、最も厳しいラベルです。 [C5](#c5)、 [C6](#c6)、 [C7が含まれます](#c7)。

#### C5 {#c5}

関心に基づくターゲティング（パーソナライゼーション）は、次の3つの条件が満たされた場合に発生します。 (1)ユーザの興味を推測するために、(2)を他のサイトやアプリ（オフサイト）などの別のコンテキストで使用し、(3)を使用して、それらの推測に基づいて提供されるコンテンツや広告を選択します。

オンサイトデータとオフサイトデータの組み合わせ、または複数のオフサイトソースのデータの組み合わせを含む、複数のサイトのデータの組み合わせをクロスサイトデータと呼びます。 サイトが異なると、異なるコンテキストが表され、クロスサイトデータのコンテキストでの使用が元のデータと異なります。 通常、クロスサイトデータは収集および処理され、ユーザーの興味に関する推測が行われます。 その結果、広告やコンテンツをターゲットにするためのクロスサイトデータの使用は、通常、広告やコンテンツがオンサイトに表示されるかオフサイトに表示されるかに関係なく、関心に基づくターゲティングの対象となります。 例えば、オンサイトデータをオフサイトデータと組み合わせて使用して、組織のサイトでユーザーを表示する広告を選択した場合、その使用は、関心に基づくターゲティングの対象と見なされます。 別の例として、オフサイトのユーザーに広告をリターゲティングする場合も、関心に基づくターゲティングと見なされる可能性が高くなります。

オフサイトのデータは通常、ユーザーの興味に関する推論を行うために収集され処理されるので、ターゲット設定にオフサイトのデータを単独で使用する場合も、関心ベースのターゲティングと見なされる可能性が高くなります。

ただし、オンサイトデータのみを使用するコンテンツや広告をターゲット設定した場合、通常、関心ベースのターゲット設定とは見なされません。 他の方法では関心に基づくターゲティングとは見なされないオンサイトターゲティングは、2つの異なるラベルとして扱われます。 特に、ラベルC6はオンサイト広告のターゲット設定とレポートに対応し、特に広告の選択、配信、レポートに対応し、ラベルC7はオンサイトコンテンツの選択、配信、レポート（オンサイトコンテンツのターゲット設定）に対応します。

最終的に、ラベルの解釈と、そのラベルを使用するデータの使用の強制は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB: パーソナライズ。 他のWebサイトやアプリなどの他のコンテキストで、広告やコンテンツを時間の経過と共にパーソナライズするために、このサービスの使用に関する情報の収集と処理。 通常、サイトやアプリのコンテンツは、広告やコンテンツの将来の選択に関する興味を推測するために使用されます。

DAA: オンライン行動広告。 特定のコンピュータまたはデバイスから、時間とアフィリエイト以外のWebサイトに関するデータを収集し、そのデータを使用して、ユーザーの好みや興味を予測し、そのWeb表示行動から推測される好みや興味に基づいて広告を配信する。

#### C6 {#c6}

広告とは、主に商品やサービスの販売を促進することを目的としたWebサイトやアプリに表示される、テキストや画像を含むメッセージや通知のことです。 このようなメッセージや通知の目的は、ユーザー次第です。 広告はオンサイトコンテンツとは別のもので、ラベル [C7が適用されます](#c7)。 C6ラベルを持つデータは、組織のWebサイトやアプリに対する広告の選択と配信、またはその広告の配信と効果の測定など、オンサイト広告のターゲット設定には使用できません。 これには、以前に収集したユーザの興味に関するオンサイトデータを使用して広告を選択し、広告が表示された日時と場所に関する処理データ、広告のクリックや購入など、広告に関するアクションをユーザが行ったかどうかが含まれます。 一般に、ユーザーのオンサイトアクティビティに基づくユーザーの環境設定を行い、オンサイト広告ターゲットでこれらの環境設定を使用しても、関心に基づくターゲティングに必要な3つの要件すべてを満たさないので、関心に基づくターゲティング（パーソナライゼーション）とは見なされません。 _[これらの要件については、label C5を参照してください。](#c5)_

最終的に、ラベルの解釈と、そのラベルを使用するデータの使用の強制は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB: 3. 広告の選択、配信、レポート: 情報の収集、および以前に収集した情報との組み合わせ、広告を選択して配信し、その広告の配信と有効性を測定する。 これには、以前に収集した関心事に関する情報を使用して広告を選択し、表示された広告に関するデータ、表示された広告の頻度、日時、場所、広告のクリックや購入など、広告に関するアクションを実行したかどうかの処理が含まれます。 これには、パーソナライゼーションは含まれません。パーソナライゼーションとは、Webサイトやアプリなど他のコンテキストで広告やコンテンツを時間の経過と共にパーソナライズするために、このサービスの使用に関する情報を収集し、処理するものです。

DAA: オンライン行動分析広告には、ファーストパーティ、広告配信、広告レポート、コンテキスト広告(訪問中のウェブページの内容に基づく広告、訪問中のウェブページへの訪問、検索クエリ)のアクティビティは含まれません。

#### C7 {#c7}

オンサイトコンテンツとは、商品やサービスの販売を促進する目的で作成されたものではなく、情報の伝達、教育、接待を目的としたテキストや画像を指します。 コンテンツがネイティブ広告と見なされるかどうかなど、コンテンツの目的はユーザーが決定します。 C7のラベルは、オンサイト広告をカバーするものではなく、ラベル [C6がカバーします](#c6)。 C7ラベルの付いたデータは、組織のWebサイトやアプリでのコンテンツの選択や配信、そのようなコンテンツの配信や有効性の測定など、オンサイトコンテンツのターゲティングには使用できません。 これには、選択したコンテンツに対するユーザーの関心、表示されたコンテンツに関するデータの処理、表示されたコンテンツの頻度や表示時間、表示日時や表示場所、コンテンツのクリックなど、コンテンツに関連する操作を行ったかどうかに関する情報が含まれます。 一般に、ユーザーのオンサイトアクティビティに基づいてユーザーの環境設定を行い、それらの環境設定をオンサイトコンテンツターゲティングで使用した場合、関心に基づくターゲティングに必要な3つの要件すべてを満たさないので、関心に基づくターゲティングとは見なされません。 _[これらの要件については、label C5を参照してください。](#c5)_

最終的に、ラベルの解釈と、そのラベルを使用するデータの使用の強制は、ユーザー次第です。 参照用に、IABおよびDAAフレームワークを次に示します。

IAB: 4. コンテンツの選択、配信、レポート: 情報の収集、および以前に収集した情報との組み合わせ、コンテンツの選択と配信、およびそのようなコンテンツの配信と有効性の測定。 これには、以前に収集した関心事に関する情報を使用したコンテンツの選択、コンテンツの表示回数、表示時間、表示日時、表示場所、コンテンツのクリックなど、コンテンツに関連するアクションを実行したかどうかのデータの処理が含まれます。 これには、パーソナライゼーションは含まれません。パーソナライゼーションとは、Webサイトやアプリなど他のコンテキストでユーザーに向けたコンテンツや広告を、本サービスの使用に関する情報を収集し、処理することです。

DAA: オンライン行動分析広告には、ファーストパーティ、広告配信、広告レポート、コンテキスト広告(訪問中のウェブページの内容に基づく広告、訪問中のウェブページへの訪問者、検索クエリ)のアクティビティは含まれません。

#### C8 {#c8}

組織のサイトやアプリのユーザーの使用状況を測定、理解、レポートする目的では、データを使用できません。 これには、関心に基づくターゲティング（クロスサイトターゲティング）は含まれません。関心に基づくターゲティング（クロスサイトターゲティング）は、その後Webサイトやアプリなど他のコンテキストでコンテンツや広告をパーソナライズするための、このサービスの使用に関する情報の集まりです。

#### C9 {#c9}

一部の契約には、データ科学のためのデータ使用に対する明示的な禁止が含まれます。 人工知能(AI)、機械学習(ML)、モデリングでのデータの使用を禁止する用語で表現されることもあります。

#### C10 {#c10}

一部のデータ使用ポリシーは、パーソナライゼーションのためのステッチIDデータの使用を制限します。 C10のラベルは、セグメントの結合ポリシーで「プライベートグラフ」オプションが使用されている場合に、セグメントに自動的に適用されます。
