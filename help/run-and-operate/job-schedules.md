# ジョブスケジュールの検査

>[!AVAILABILITY]
>
>[!UICONTROL Job schedules] は現在、限定リリースとして、次のReal-Time CDP ジョブでのみ利用できます。
>
> * バッチデータレイクの取り込み
> * バッチプロファイル取り込み
> * バッチ送信
> * バッチ宛先のアクティベーション。

[!UICONTROL Job Schedules] は、取り込みから宛先のアクティベーションまで、データパイプライン全体にわたるすべてのスケジュール済みバッチ処理ジョブの統合ビューを提供します。 実行ステータスを検査し、スケジュールの競合を特定し、ビジネス運用に影響を与える前に設定の問題を診断します。

ジョブスケジュールを使用して、エラーを調査し、ジョブのタイミングを最適化し、データレイクの取り込み、プロファイル処理、セグメント化および宛先アクティベーション間の依存関係を理解します。 一般的な設定の問題を解決する方法については、[ ジョブスケジュールのアンチパターンの特定 ](job-schedules-anti-patterns.md) に関するドキュメントを参照してください。

## 前提条件 {#prerequisites}

[!UICONTROL Job Schedules] にアクセスするには、**[!UICONTROL View Job Schedules]** と **[!UICONTROL View Profile Management]** [ アクセス制御権限 ](/help/access-control/home.md#permissions) が必要です。

システム管理者に問い合わせて、適切な権限があることを確認します。

## はじめに {#getting-started}

[!UICONTROL Job Schedules] を使用する前に、次のExperience Platformの概念を理解しておく必要があります。

* **[バッチ取り込み](../ingestion/batch-ingestion/overview.md)**：スケジュールされた間隔でデータレイクとプロファイルストアにデータを読み込む方法。
* **[セグメント化](../segmentation/home.md)**：プロファイルデータとセグメント定義に基づいて、オーディエンスを評価および更新する方法。
* **[リアルタイム顧客プロファイル](../profile/home.md)**：プロファイルデータを統合してセグメント化やアクティベーションに使用できるようにする方法。
* **[宛先](../destinations/home.md)**：ダウンストリームシステムおよびマーケティングプラットフォームに対してデータをアクティブ化する場所と方法。

これらのコンポーネントを理解することで、ジョブ実行パターンを解釈し、発生時に問題を診断するのに役立ちます。

## ジョブスケジュールインターフェイスについて {#understanding-interface}

[!UICONTROL Job Schedules] にアクセスするには：

1. Experience Platform UI で、左側のナビゲーションから「**[!UICONTROL Run and Operate]**」を選択します。
2. **[!UICONTROL Job Schedules]** を選択します。

[!UICONTROL Job Schedules] のページには、スケジュールされたすべてのバッチ処理ジョブの概要が表示されます。

![ 左側のナビゲーションの実行と操作 ](assets/job-schedules/run-and-operate-left-nav.png)

### 概要カード {#summary-cards}

ページの上部には、バッチ処理ジョブに関するクイックインサイトを提供する概要カードが表示されます。

![ バッチ処理ジョブに関するインサイトを示すジョブスケジュールの概要カード ](assets/job-schedules/job-schedules-cards.png)

* **レイクの取り込み実行数**：実行されたデータレイクの取り込みジョブの数。
* **プロファイル取り込み実行数**：実行されたプロファイル取り込みジョブの数。
* **次のセグメント化**：次にスケジュールされたセグメント化ジョブがいつ実行されるか。
* **次回の宛先アクティベーション**：次にスケジュールされた宛先アクティベーションジョブがいつ実行されるかを示します。

これらのカードは、データパイプライン全体のアクティビティと今後のスケジュールを理解するのに役立ちます。 **レイクの取り込み実行** および **プロファイルの取り込み実行** の値は、選択した時間間隔（今日、昨日または過去 7 日間）に基づいて変化します。次回実行されるカード（**次のセグメント化** および **次の宛先のアクティベーション**）は、時間セレクターの影響を受けません。

### 期間セレクター {#time-period}

期間セレクターを使用して、スケジュールされたジョブを表示するまでの時間を選択します。

![ ジョブスケジュールの期間セレクター UI のアニメーションの例 ](assets/job-schedules/time-selector.gif)

* **Today**：今日にスケジュールされたジョブを表示します（デフォルト）。
* **昨日**：昨日実行されたジョブを表示します。
* **過去 7 日間**：過去 1 週間のジョブを表示します。

### バッチジョブスケジュールの詳細 {#job-schedules-details}

メイン表示には、バッチジョブの実行が 1 日を通してスケジュールされている場合に表示されます。 実行できる操作は、次のとおりです。

* **データセットまたはエンティティ別にジョブを表示**：左の列には、データセットまたは処理ジョブの名前が表示されます（例：取り込みデータセットやセグメント化ジョブ）。
* **ジョブのタイミングを参照**：タイムラインは各ジョブの実行がスケジュールされた日時を表示し、視覚的な指標がスケジュールされた時間を示します。
* **フィルタージョブ**：フィルターアイコンを使用して、レポートに含めるデータセットを絞り込みます。
* **ジョブタイプについて**：下部に色分けされた凡例を使用すると、様々なジョブタイプを識別できます。
   * **レイクの取り込み** （緑）：データレイクへのデータ取り込み
   * **プロファイル取り込み** （ピンク）：プロファイルストアへのデータ取り込み
   * **セグメント化** （水色）：オーディエンス評価ジョブ
   * **プロファイルの書き出し** （青）：プロファイルデータの書き出し
   * **アクティベーション** （ダークグレー）：宛先アクティベーションジョブ
   * **処理中** （ストライプ）：現在実行中またはキュー内にあるジョブ

このタイムライン表示は、スケジュールの競合の特定、ジョブ間の依存関係の理解、バッチ処理スケジュールの最適化に役立ちます。

## 設定の問題の特定 {#identifying-issues}

ジョブスケジュールを確認すると、設定の問題を示すパターンに気付くことがあります。 一般的な問題を次に示します。

* ジョブのスケジュールが近すぎると、リソースの競合が発生します
* 同じ時間枠内で実行されるバッチが多すぎます
* 過剰な毎日のバッチジョブを含む個々のデータセット
* セグメント化の実行直前にスケジュールされた取り込みジョブ

これらのパターンは、ジョブの失敗、不完全なデータ処理、システムパフォーマンスの低下につながる可能性があります。 これらの問題を特定し解決する方法については、[ ジョブスケジュールのアンチパターンの特定 ](job-schedules-anti-patterns.md) に関するドキュメントを参照してください。

特定のデータセットやジョブ実行を調査する必要がある場合は、詳細ビューにドリルダウンして、実行履歴、エラーメッセージ、パフォーマンス指標および依存関係を確認できます。 この詳細なデータの表示について詳しくは、[ ジョブの詳細の表示 ](job-schedules-details.md) に関するドキュメントを参照してください。
