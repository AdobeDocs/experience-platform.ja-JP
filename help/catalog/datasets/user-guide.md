# Datasetsユーザーガイド

このユーザーガイドでは、Adobe Experience Platformユーザーインターフェイス内でデータセットを操作する際に、一般的なアクションを実行する手順を説明します。

## はじめに

このユーザーガイドでは、Adobe Experience Platformの次のコンポーネントについて、実際に理解している必要があります。

* [データセット](overview.md):Experience Platformのデータ永続化のためのストレージと管理の構成。
* [Experience Data Model(XDM)System](../../xdm/home.md):エクスペリエンスプラットフォームが顧客エクスペリエンスデータを整理するための標準化されたフレームワーク。
   * [スキーマ構成の基本](../../xdm/schema/composition.md):XDMスキーマの基本的な構成要素について説明します。この中には、主な原則や構成のベストプラクティスが含まれています。スキーマ構成の基本要素です。
   * [スキーマエディタ](../../xdm/tutorials/create-schema-ui.md):Platformユーザーインターフェイス内のスキーマエディターを使用して、独自のカスタムXDMスキーマを構築する方法を説明します。
* [リアルタイム顧客プロファイル](../../profile/home.md):複数のソースからの集計データに基づいて、統合されたリアルタイムのプロファイルを顧客に提供します。
* [データガバナンス](../../data-governance/home.md):お客様のデータの使用に関する規制、制限、ポリシーへの準拠を確保します。

## リアルタイム顧客データセットの有効化プロファイル

すべてのデータセットには、取り込んだデータを顧客プロファイルに拡張する機能があります。 そのためには、データセットが準拠するスキーマが、リアルタイム顧客プロファイルでの使用に対して互換性がある必要があります。 互換性のあるスキーマは、次の要件を満たします。

* スキーマに、identityプロパティとして指定された属性が1つ以上あります。
* スキーマには、プライマリIDとして定義されたIDプロパティがあります。

プロファイル用のユーザーを有効にする方法について詳しくは、 [スキーマエディターのユーザーガイドを参照してくださ](../../xdm/tutorials/create-schema-ui.md)い。

プロファイル用のデータセットを有効にするには、そのデータセ *ットのアクティビティ* 画面にアクセスし、「プロパテ **ィ** 」列内のプロファイル切り替えをクリック *します* 。 有効にすると、データセットに取り込まれたデータを使用して顧客のデータを入力することもプロファイルします。

![](../images/datasets/user-guide/enable_dataset_profiles.png)

データセットに既にデータが含まれ、その後プロファイルが有効になっている場合、既存のデータはプロファイルによって消費されません。 データセットのプロファイルを有効にした後、既存のデータを再度取り込み、顧客のプロファイルを設定することをお勧めします。

## データセットのデータガバナンスの管理と実施

Data Usage Labeling and Enforcement(DULE)は、エクスペリエンスプラットフォームの中核的なデータガバナンスメカニズムです。 DULEラベルを使用すると、データに適用される使用ポリシーに従ってデータセットとフィールドを分類できます。 ラベルの詳細につ [いては、『](../../data-governance/home.md) Data Governanceの概要 [』を参照してください。また、データセットにラベルを適用する方法については、『](../../data-governance/labels/overview.md) Data usage labels』ユーザーガイドを参照してください。

## データセットの削除

データセットを削除するには、まずデータセットのアクティビティ画面に *アクセスし* ます。 次に、「データセット **の削除** 」をクリックして削除します。

>[!NOTE] アドビのアプリケーションおよびサービス（Adobe Analytics、Adobe Decisioning Manager、Decisioning Serviceなど）で作成および利用されたデータセットは削除できません。

![](../images/datasets/user-guide/delete_dataset.png)

確認ボックスが表示されます。 「削除」 **をクリックし** 、データセットの削除を確定します。

![](../images/datasets/user-guide/confirm_delete.png)

## プロファイル対応データセットの削除

データセットのプロファイルが有効になっている場合、UIからデータセットを削除すると取り込み用のデータセットが無効になりますが、バックエンドのデータセットは自動的には削除されません。 データセットに含まれるプロファイルとIDデータを完全に削除するには、追加の削除リクエストを行う必要があります。 プロファイルストアからデータを適切に削除する手順については、プロファイルシステムジョブ（「削除要求」とも呼ばれる）のReal-time Customer [Commenter APIサブガイドを参照してください](../../profile/api/profile-system-jobs.md)。

## データの取り込みの監視

エクスペリエンスプラットフォームUIで、左側のナ **ビゲーション** 「監視」をクリックします。 監視 *ダッシュボードを使用すると* 、バッチまたはストリーミング取り込みから受信データのステータスを表示できます。 個々のバッチのステータスを表示するには、「エンドツーエ *ンドのバッチ処理」または* 「エンドツーエ *ンドのストリーミング」をクリックします*。 ダッシュボードは、正常、失敗、または進行中のバッチまたはストリーミングの取り込みを含む、すべてのバッチまたはストリーミングの取り込みをリストします。 各リストには、バッチID、ターゲットデータセットの名前、取り込んだレコード数など、バッチの詳細が表示されます。 ターゲットデータセットのプロファイルが有効になっている場合は、取り込んだIDレコードとプロファイルレコードの数も表示されます。

![](../images/datasets/user-guide/batch_listing.png)

個々の **Batch ID** （バッチID）をクリックして *Batch overview* （バッチの概要）ダッシュボードにアクセスし、バッチの取り込みに失敗した場合にエラーログを含むバッチの詳細を確認できます。

![](../images/datasets/user-guide/batch_overview.png)

バッチを削除する場合は、ダッシュボードの右上にある「 **Delete batch** found」をクリックします。 また、バッチが最初に取り込まれたデータセットからもレコードが削除されます。

![](../images/datasets/user-guide/delete_batch.png)
