* オプションで、「説明」フィールドを使用してスキーマに関する追加情報を **入力できま** す。

## スキーマフィールドをIDフィールドとして設定する {#identity-field}

スキーマは、データをエクスペリエンスプラットフォームに取り込むために使用され、最終的には、個人を識別し、複数のソースからの情報を組み合わせるために使用されます。 この処理を支援するために、キーフィールドを「ID」フィールドとしてマークできます。

Experience Platformでは、スキーマエディターの「ID」チェックボックスを使用して、IDフ **ィールド** を簡単に示します。

例えば、同じ「レベル」に属する忠誠度プログラムのメンバーが何千人もいる場合がありますが、忠誠度プログラムの各メンバーは一意の「忠誠度ID」（この場合は、個々のメンバーの電子メールアドレス）を持ちます。 「loyaltyId」が各メンバーの一意の識別子であることは、IDフィールドに適した候補となりますが、「level」は異なります。

## スキーマをリアルタイム顧客プロファイル {#profile}

スキーマエディターを使用すると、スキーマをリアルタイム顧客 [プロファイルで使用できます](../../profile/home.md)。 プロファイルは、顧客属性の360°の堅牢なプロファイルと、エクスペリエンスプラットフォームに統合されたシステム全体での顧客のすべてのインタラクションのタイムスタンプ付きのアカウントを構築することで、各顧客の全体的な表示を提供します。

スキーマをリアルタイム顧客プロファイルで使用できるようにするには、プライマリIDが定義されている必要があります。 最初にプライマリIDを定義せずにスキーマを有効にしようとすると、「プライマリIDが見つかりません」というエラーメッセージが表示されます。

![](../images/tutorials/create-schema/missing_primary_identity.png)

プロファイルでの「ロイヤルティメンバー」スキーマの使用を有効にするには、エディターの「構造」セクションで「ロイヤリティメ *ンバー* 」をクリックします。

エディタの右側の[ *スキーマのプロパティ*]に、スキーマの表示名、説明、タイプなどの情報が表示されます。 この情報に加えて、「 **プロファイル」というトグルボタン**。

![](../images/tutorials/create-schema/unified_profile_toggle.png)

「 **プロファイル** 」をクリックすると、プロファイル用のスキーマを有効にするかどうかを確認するポップアップが表示されます。

![](../images/tutorials/create-schema/enable_unified_profile.png)

>[!NOTE] スキーマをリアルタイム顧客プロファイルに対して有効にし、保存すると、無効にできません。

## 次の手順

「忠誠度メンバー」スキーマの構成が完了すると、エディターの「構造」セクションに完全な *スキーマ* が表示されます。 「保存 **** 」をクリックすると、スキーマがスキーマライブラリに保存され、スキーマレジストリからアクセスできるようになります。

これで、新しいスキーマを使用して、データをプラットフォームに取り込むことができます。 データの取り込みにスキーマを使用した後は、追加的な変更のみが行われる場合があります。 バージョン管 [理について詳しくは](../schema/composition.md) 、スキーマ構成の基本を参照してください。

「忠誠度メンバー」スキーマは、スキーマレジストリAPIを使用して表示および管理することもできます。 APIの使用を開始するには、開始レジストリAPI開発ガ [イドを読んでスキーマします](../api/getting-started.md)。

### Create a new class {#create-new-class}

エクスペリエンスプラットフォームは、組織に固有のスキーマに基づいてプラットフォームを柔軟に定義できる機能です。

クラスの割り当 *て(* Assign Class **)ダイアログを開くには、** スキーマエディタの「クラス ** 」セクションで「割り当て」をクリックします。 ダイアログ内で、[新しいクラ **スの作成]を選択します**。

### クラスの変更スキーマ {#change-class}

最初のスキーマ構成プロセス中は、スキーマを保存する前に、いつでもスキーマの基となるクラスを変更できます。

>[!WARNING] 授業を変える前に注意してください。 ミックスインは特定のクラスとのみ互換性があるので、クラスを変更するとキャンバスがリセットされ、その時点に追加したフィールドが削除されます。

クラスを変更するには、エディタの **** Composition *（組版）セクションでClass* （クラス） *の横のAssign* （割り当て）をクリックします。

[クラスの割 *り当て* ]ダイアログが開いたら、使用可能なオプションから新しいクラスをリストします。 「クラ **スを割り当て** 」をクリックすると、新しいクラスを割り当てるかどうかを確認する新しいダイアログが開きます。

![クラスの変更](../images/tutorials/create-schema/assign_new_class_warning.png)

クラスの変更を確認すると、キャンバスがリセットされ、構成の進行状況がすべて失われます。
