# 見込み顧客の使用例を通じて新規顧客を惹きつけ、獲得する

>[!AVAILABILITY]
>
>* このベータ版機能は、Real-Time CDP（アプリサービス）、Adobe Experience Platform Activation、リアルタイム CDP、Real-Time CDP Prime、Real-Time CDP Ultimate のライセンスを持つお客様が利用できます。 これらのパッケージの詳細については、 [製品の説明](https://helpx.adobe.com/legal/product-descriptions.html) 詳しくは、Adobe担当者にお問い合わせください。

Real-Time CDPのサードパーティのデータサポートを利用して、データパートナーの見込み客プロファイルを活用してプロファイルベースを拡張し、新しい顧客を獲得またはリーチするためにデータパートナーと連携します。

![顧客見込みユースケースの概要を視覚的に説明します。](/help/rtcdp/assets/partner-data/prospecting/prospecting-use-case-overview.png)

## 前提条件と計画 {#prerequisites-and-planning}

Real-Time CDPのパートナーデータサポートを利用して新しい顧客に連絡し、獲得する際は、計画プロセスで次の前提条件を考慮します。

* パートナー提供のプロファイルをReal-Time CDPに取り込んで更新する際のサイクルは何ですか？
* ダウンストリームの宛先で必要な ID は何ですか？
* 取り込まれる識別子がアクションにつながるダウンストリームであることを確認します。
* 取り込むパートナーデータは、広く受け入れられた永続的な識別子 ( 個人識別情報 (PII)、ハッシュ化された PII、パートナー識別子など ) に関連付けられていますか？
* パートナーの観点や、自社の法務、プライバシー、コンプライアンスチームから、どのようなデータ使用ポリシーに注意する必要がありますか。

## このユースケースの達成方法：概要 {#achieve-the-use-case-high-level}

Real-Time CDPを拡張して新しい顧客を惹きつけ、獲得する前に、Real-Time CDPを使用してファーストパートデータの堅牢な基盤を構築してください。 この使用例を達成するワークフローは、既知の顧客と関わるワークフローに似ています。

![顧客見込みユースケースの概要を視覚的に説明します。](/help/rtcdp/assets/partner-data/prospecting/prospecting-use-case-steps.png)

1. As a **顧客**、1 つ以上の見込み客プロファイルのライセンスを取得しています **データパートナー** を使用して、ファネルの顧客獲得を促進できます。
2. As a **顧客**&#x200B;を使用する場合は、プロファイルデータとガバナンスモデルを拡張して、 **パートナー** — 見込み客プロファイルのリストが提供されます。
3. As a **顧客**&#x200B;では、見込み客のプロファイルをReal-Time CDPに読み込み、ガバナンスポリシーを構築して、責任を持って使用するようにします。
4. As a **顧客**&#x200B;を使用すると、見込み客プロファイルのリストから焦点を絞ったオーディエンスを構築できます。
5. As a **顧客**&#x200B;を使用して、見込み客オーディエンスを、見込み客リストで使用可能な id を受け入れる宛先に対してアクティブ化します。
6. 必要に応じて、 **データパートナー** ：対象の有料メディアの宛先に対するオーディエンスのラストマイルアクティブ化。

### パートナーからサードパーティのプロファイルの詳細をライセンス取得 {#license-profiles-from-partner}

### パートナーが提供する見込み客のプロファイルに対応するよう、プロファイルデータとガバナンスモデルを拡張 {#extend-governance-model}

見込み客のプロファイルをデータパートナーから受け取る準備をするには、Real-Time CDPのデータ管理フレームワークを拡張して、この新しいプロファイルタイプに対応する必要があります。

使用する ID、データ管理、ガバナンスの各コンポーネントは、次のとおりです。

* 新しい **[!UICONTROL パートナー ID]** パートナー提供プロファイルの ID タイプ
* 新しい **[!UICONTROL XDM 個別見込み客プロファイル]** XDM クラス
* **（ドキュメントは近日公開予定）** パートナーデータサポートに合わせてカスタマイズされたフィールドグループ
* **（ドキュメントは近日公開予定）** パートナーから取得される属性に追加するサードパーティラベル

#### パートナー ID 名前空間の作成

まず、パートナーから受け取るプロファイルの新しい ID タイプを作成します。 これをおこなうには、「ID」セクションで、タイプの新しい ID 名前空間を作成する必要があります **[!UICONTROL パートナー ID]**.

![新しいパートナー ID 名前空間を作成します。](/help/rtcdp/assets/partner-data/prospecting/create-partner-identity-namespace.png)

* 詳しくは、 [id タイプセクション](/help/identity-service/namespaces.md).
* 詳細 [ID フィールドの定義方法](/help/xdm/ui/fields/identity.md) 」をクリックします。

#### を使用して新しいスキーマを作成します。 **[!UICONTROL XDM 個別見込み客プロファイル]** クラス

次へ、 **[!UICONTROL データ管理]** > **[!UICONTROL スキーマ]**、新しいスキーマを作成し、 **[!UICONTROL XDM 個別見込み客プロファイル]** クラス。

![XDM スキーマビルダーで XDM 個別見込み客プロファイルクラスを検索します。](/help/rtcdp/assets/partner-data/prospecting/xdm-individual-prospect-class.png)

方法を読む [UI でのスキーマの作成と編集](/help/xdm/ui/resources/schemas.md) XDM 個別見込み客プロファイルクラスに関する完全な情報を取得します（今後のリンク）。

この **[!UICONTROL XDM 個別見込み客プロファイル]** クラスには、次に示すフィールドが事前設定されています。 見込み客プロファイルに対してパートナーが指定した属性でスキーマをエンリッチメントするには、期待する属性で新しいフィールドグループを作成してAdobeに追加するか、スキーマが提供する事前設定済みのフィールドグループの 1 つを使用します。

![XDM 個別見込み客プロファイルクラスの事前設定済みフィールド。](/help/rtcdp/assets/partner-data/prospecting/preconfigured-fields-individual-prospect-class.png)

次に、前の手順で作成した partnerID ID を、スキーマのプライマリ ID として選択する必要があります。 プロファイルレコードには主な識別子が含まれている必要があります。 この手順は、見込み客データをプロファイルストアに読み込み、セグメント化とアクティブ化に使用できるようにするために必要です。

>[!AVAILABILITY]
>
>見込み客プロファイルは、自動的に、パートナー ID タイプの ID 名前空間のみを使用するように制限されます。 これは設計に基づいており、ファーストパーティプロファイルと見込み客プロファイルを確実に分離できます。

![以前に設定したパートナー ID を、スキーマのプライマリ ID として選択します。](/help/rtcdp/assets/partner-data/prospecting/select-partner-id-as-primary-identity.gif)

スキーマは、まだプロファイルに対して有効になっていません。 「プロファイル」ボタンを切り替えて、このスキーマをプロファイルサービスに含めるように設定します。 リアルタイム顧客プロファイルでスキーマの使用を有効にする方法について詳しくは、 [スキーマの作成に関するチュートリアル](/help/xdm/tutorials/create-schema-ui.md#profile)

![プロファイルでスキーマを有効にする.](/help/rtcdp/assets/partner-data/prospecting/enable-schema-for-profile.png)

#### スキーマ内のすべてのフィールドにサードパーティのデータガバナンスラベルを追加する

スキーマを構成するすべてのフィールドに、サードパーティのデータガバナンスラベルを追加することを検討します。 これは、サードパーティのデータを責任を持って使用し、データ漏洩のリスクを最小限に抑えるために重要です。 サードパーティのデータガバナンスラベルの詳細を見る（Jordan によるドキュメントへのリンクを追加）

これを行うには、以下の手順に従います。

1. 作成したスキーマに移動し、 **[!UICONTROL ラベル]** タブをクリックします。
2. 上部のチェックボックスボタンを使用してこのスキーマ内のすべてのフィールドを選択し、右側の鉛筆アイコンをクリックして、このスキーマにデータガバナンスラベルを適用します。
3. を選択します。 **[!UICONTROL パートナーエコシステム]** 左側のカテゴリのラベル。
4. というラベルを選択します。 **[!UICONTROL サードパーティ]** を選択し、 **[!UICONTROL 保存]**.
5. スキーマ内のすべてのフィールドに、前の手順で選択したラベルが含まれるようになります。

>[!SUCCESS]
>
>これで、スキーマを使用する準備が整い、次の手順に進み、見込み客データをデータパートナーから取り込むことができます。

また、この手順では、データ管理戦略を拡大して、パートナーから提供されるサードパーティのデータを含める際の、データガバナンスモデルの変化について考えます。 以下のドキュメントリンクにある考慮事項を参照してください。

* (**近日開始**) サードパーティデータを別のデータセットに保持して、統合の削除と取り消しを容易におこなえるようにします。
* (**近日開始**) [データセットの有効期限](/help/hygiene/ui/dataset-expiration.md) の機能は、データ衛生アドオンを購入したクライアントのデータセットに対して使用できます。
* (**近日開始**) サードパーティデータを取り込む派生データセットを作成する場合は注意が必要です。混在させた場合、サードパーティデータを削除する唯一の方法は、派生データセット全体を削除することです。

### 見込み客プロファイルのリストを読み込み、見込み客プロファイルビューを調べます

見込み客プロファイルを管理するためのデータモデルを用意したら、次にデータを取り込みます。

#### データセットを作成し、サンプルの見込み客データを読み込みます

サンプルデータを読み込み、見込み客のプロファイルを設定するには、データセットを作成し、データパートナーから受け取ったファイルをアップロードします。 以下の手順を実行します。

1. に移動します。 **[!UICONTROL データ管理]** > **[!UICONTROL データセット]** を選択し、 **[!UICONTROL データセットを作成]**.
2. スキーマからデータセットを作成を選択
3. 前の手順で作成したスキーマを選択します。
4. データセットに名前を付け、必要に応じて説明を追加します。
5. 「**[!UICONTROL 完了]**」を選択します。

![見込み客プロファイルのデータセットを作成する手順の記録。](/help/rtcdp/assets/partner-data/prospecting/create-dataset-for-prospect-profiles.gif)

スキーマを作成する手順と同様に、データセットをリアルタイム顧客プロファイルに含める必要があります。 リアルタイム顧客プロファイルでデータセットを使用する方法について詳しくは、 [スキーマの作成に関するチュートリアル](/help/xdm/tutorials/create-schema-ui.md#profile)

![プロファイルのデータセットを有効にします。](/help/rtcdp/assets/partner-data/prospecting/enable-dataset-for-profile.png)

パートナーから受け取ったファイルをデータセットに読み込むには、データセットを選択し、右側のパネルで下にスクロールして、「 」を選択します。 **[!UICONTROL データを追加]**. ファイルをドラッグ&amp;ドロップするか、 **[!UICONTROL ファイルを選択]** をクリックしてファイルの場所に移動し、選択します。

![データセットにファイルを追加します。](/help/rtcdp/assets/partner-data/prospecting/add-file-to-dataset.png)

データパートナーからReal-Time CDPにプロファイルのリストを読み込んだ後、 [Inspect読み込まれた見込み客プロファイルセクション](#inspect-profiles) をクリックして、見込み客のプロファイルが UI に入力されていることを確認します。

#### ソースコネクタを使用した見込み客データの取り込み

繰り返しのファイルインポートを設定して、ソースコネクタを通じてパートナーからデータを取り込み、見込み客プロファイルのリストをReal-Time CDPに取り込むことができます。

この目的で推奨されるソースコネクタの一部を以下に示しますが、Real-Time CDPへのファイルベースの取り込み方法は、お客様の目的で機能することに注意してください。

* [Amazon S3](/help/sources/connectors/cloud-storage/s3.md)
* [SFTP](/help/sources/connectors/cloud-storage/sftp.md)

データパートナーからReal-Time CDPにプロファイルのリストを読み込んだ後、次の節に進み、見込み客のプロファイルが UI に入力されていることを確認します。

#### Inspect読み込まれた見込み客プロファイル {#inspect-profiles}

見込み客プロファイルのリストを確認するには、に移動します。 **[!UICONTROL 見込み客]** > **[!UICONTROL プロファイル]** をクリックします。

Real-Time CDPに読み込んだ見込み客プロファイルが **[!UICONTROL 参照]** 見込み客プロファイル画面のビュー。 ページに「現在、閲覧する見込み客のプロファイルがありません」というメッセージが表示される場合は、しばらくしてからもう一度試してください。 しばらく待った後、見込み客プロファイルが **[!UICONTROL 参照]** 表示

>[!TIP]
>
>この **[!UICONTROL ID 名前空間]** 列。 複数のデータベンダーと連携している場合は、この列を使用して、見込み客プロファイルの起源を推測します。

![Real-Time CDPに読み込まれた見込み客プロファイルの表示。](/help/rtcdp/assets/partner-data/prospecting/prospect-profiles-view.png)

次に示すように、任意の見込み客プロファイルを選択して、さらに検査することもできます。

![見込み客プロファイルの検査方法のビュー。](/help/rtcdp/assets/partner-data/prospecting/inspect-prospect-profile.gif)

(**近日開始**) 見込み客プロファイルの詳細を表示します。

### 見込み客オーディエンスの作成 {#create-prospect-audiences}

Real-Time CDPのセグメント化機能を使用して、見込み客プロファイルからオーディエンスを作成します。 目的のセグメント化ルールを使用して、カスタマイズしたオーディエンスを作成します。

見込み客プロファイルで構成されたオーディエンスを開始および構築するには、以下に移動します。 **[!UICONTROL 見込み客]** > **[!UICONTROL オーディエンス]**.

![見込み客オーディエンスの表示。](/help/rtcdp/assets/partner-data/prospecting/prospect-audiences.png)

見込み客プロファイルのオーディエンス構築エクスペリエンスは、既知のファーストパーティ顧客からオーディエンスを構築するエクスペリエンスとは異なることに注意してください。 この表示は次の項目に制限されます。

* 先ほど作成した見込み客クラスの属性。
* バッチプロファイル評価のみ。
* は、時系列イベントに基づくオーディエンスの作成をサポートしていません。

(**近日開始**) 見込み客オーディエンスの詳細を表示します。

### 宛先への見込み客プロファイルのアクティブ化 {#activate-to-destinations}

見込み客オーディエンスを宛先に書き出して利用します。 現在、 [Amazon S3](/help/destinations/catalog/cloud-storage/amazon-s3.md) または [!BADGE アルファ]{type=Informative}[LiveRamp](/help/destinations/catalog/advertising/liveramp.md) 宛先は、見込み客プロファイルのアクティブ化をサポートします。
