# 結合ポリシーの概要

Adobe Experience Platform では、複数のソースからデータを組み合わせて、個々の顧客の全体像を把握できます。結合ポリシーとは、データを統合する場合のデータの優先順位付け方法と、[!DNL Platform] のデータを組み合わせてプロファイルの統一されたビューを作成するかを決定するために使用されるルールです。

RESTful API とExperience Platformユーザーインターフェイスを通じて、新しい結合ポリシーの作成、既存のポリシーの管理、組織のデフォルトの結合ポリシーの設定を行えます。 また、データソースに関する環境設定を指定し、結合中に競合が発生した場合にどのデータの優先順位を持つかを指定することもできます。

結合ポリシーと、結合ポリシーがポリシー内で果たす役割について詳しくは、まず [ 結合Experience Platformの概要 ](../../profile/merge-policies/overview.md) をお読みください。
