# [!DNL Privacy Service] トラブルシューティングガイド

Adobe エクスペリエンスプラットフォーム [!DNL Privacy Service] には、カスタマーデータのプライバシー要求を管理するための RESTFUL API およびユーザーインターフェイスが用意されています。 で [!DNL Privacy Service] は、私的な顧客データへのアクセスおよび削除要求を送信して、組織や法律上のプライバシー規制に自動的に適合させることができます。

このドキュメントでは、よくある質問への回答 [!DNL Privacy Service] 、および、API でよくあるエラーに関する情報を提供しています。

## API でプライバシーリクエストをおこなう場合、ユーザーとユーザー ID の違いは何ですか？  {#user-ids}

API で新しいプライバシージョブを作成するには、リクエストの JSON ペイロードに、プライバシーリクエストが適用される、各リストに固有の情報をする `users` 配列を含める必要があります。`users` 配列内の各項目は、`key` 値で識別される特定のユーザーを表すオブジェクトです。

次に、各ユーザーオブジェクト（または `key`）には独自の `userIDs` 配列が含まれます。この配列は、**その1 人のユーザーのために**&#x200B;特定の ID 値をリストします。

以下の `users` 配列の例について考えてみましょう。

```json
"users": [
  {
    "key": "DavidSmith",
    "action": ["access"],
    "userIDs": [
      {
        "namespace": "email",
        "value": "dsmith@acme.com",
        "type": "standard"
      }
    ]
  },
  {
    "key": "user12345",
    "action": ["access", "delete"],
    "userIDs": [
      {
        "namespace": "email",
        "value": "ajones@acme.com",
        "type": "standard"
      },
      {
        "namespace": "ECID",
        "type": "standard",
        "value":  "443636576799758681021090721276",
        "isDeletedClientSide": false
      }
    ]
  }
]
```

この配列には 2 つのオブジェクトが含まれ、それぞれの `key` 値（「DavidSmith」と「user12345」）で識別される個々のユーザーを表します。「DavidSmith」には 1 つの ID（電子メールアドレス）しかありませんが、「user12345」には 2 つ（電子メールアドレスと ECID）があります。

ユーザID情報の提供について詳しくは、[プライバシーリクエストの ID データ](identity-data.md)に関するガイドを参照してください。


## を使用し [!DNL Privacy Service] て、誤ってに送信されたデータをクリーンアップすることはでき [!DNL Platform] ますか?

アドビでは [!DNL Privacy Service] 、誤って製品に送信されたデータを削除するための使用はサポートされていません。 [!DNL Privacy Service] は、お客様が、データ主体（または消費者）のアクセスや削除のリクエストに対する義務を果たすのを支援するよう設計されています。これらのリクエストは時間に依存し、該当するプライバシー法に従っておこなわれます。データに含まれていない要求を送信すると、その要求はすべてのユーザーに影響を与えるほか、 [!DNL Privacy Service] [!DNL Privacy Service] 適切な法律タイムラインをサポートすることもできます。

PII やデータの問題をなくすための取り組みのレベルを調整および提供するには、担当のアカウントマネージャー（CDM）にお問い合わせください。

## プライバシーリクエストやジョブのステータスに関する情報び取得方法を教えてください。

特定のジョブに関する詳細を取得するには、 [!DNL Privacy Service] API またはユーザーインターフェイスを使用します。

### API の使用

API を使用して特定のジョブの状態を取得するには [!DNL Privacy Service] 、 `GET /` 要求パスにあるジョブの ID を使用して、ルート () エンドポイントに対して要求を行います。 詳しくは、「 [ ](api/privacy-jobs.md#check-the-status-of-a-job) API ガイド」の「ジョブの状態を確認する」の項を参照してください [!DNL Privacy Service] 。

### UI の使用

アクティブなジョブ要求はすべて、 **[!UICONTROL UI ダッシュボードのジョブ要求 widget に一覧表示され]** [!DNL Privacy Service] ます。 各ジョブリクエストのステータスが「**[!UICONTROL ステータス]**」列に表示されます。UI でのジョブリクエストの表示について詳しくは、[Privacy Service のユーザーガイド](ui/user-guide.md)を参照してください。

## 完了したプライバシージョブの結果をダウンロードする方法を教えてください。

[!DNL Privacy Service]API およびユーザーインターフェイスでは、完了したジョブの結果を ZIP 形式でダウンロードするためのメソッドが用意されています。

### API の使用

リクエストパスで結果をダウンロードするジョブの ID を使用して、 API のルート（`GET /`）エンドポイントにリクエストを送信します。[!DNL Privacy Service]ジョブのステータスが完了した場合、API は応答本文に `downloadURL` 属性を含めます。この属性には、ブラウザーのアドレスバーに貼り付けて ZIP ファイルをダウンロードできる URL が含まれます。

詳細については、「 [ API guide」で、その ID を使用してジョブを検索するの項を参照してください ](api/privacy-jobs.md#check-the-status-of-a-job) [!DNL Privacy Service] 。

### UI の使用

[!DNL Privacy Service]UI ダッシュボードで、ジョブ要求 widget からダウンロードするジョブを探し **** ます。「ジョブの詳細」ページを開くジョブの ID を選択します。 ここから、 **右上隅にある「ダウンロード」を選択し** て ZIP ファイルをダウンロードします。 詳細な手順については、『[Privacy Service ユーザーガイド](ui/user-guide.md)』を参照してください。

## 一般的なエラーメッセージ

| --- | --- |

| 無効な名前空間 | ユーザー ID に対して指定された id 名前空間が無効でした。 [ ](./api/appendix.md#standard-namespaces) [!DNL Privacy Service] 使用できる名前空間の一覧については、「API guide」の「標準的な id 名前空間」を参照してください。カスタム名前空間を使用している場合は、ID の `type` プロパティを &quot;custom&quot; に設定しているかどうかを確認してください。 |
| 一部が完了しました | ジョブは正常に完了しましたが、一部のデータは指定された要求に適用されずスキップされました。 |
| データが指定された形式ではありません。 | 指定されたアプリケーションの1つ以上のデータ値が正しくフォーマットされていません。 詳細については、ジョブの詳細を確認してください。 |

| アクセスおよび権限が必要です。 | を使用するためには、アクセス権と権限が必要です [!DNL Privacy Service] 。 アクセス権を取得するには、管理者に連絡してください。 |
| Access データをアップロードし、アーカイブする際に問題が発生しました。 | このエラーが発生したときに、access データを再アップロードし、もう一度やり直してください。 |
