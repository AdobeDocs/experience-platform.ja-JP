# Privacy Service エラーコード {#privacy-service-error-codes}

このリファレンスを使用して、Privacy Service ジョブの結果を特定し、エラーを診断し、**Adobe Experience Platform** でプライバシージョブを送信または監視する際の次の適切な手順を決定します。 プライバシージョブを作成、送信、監視する方法については、[&#x200B; プライバシージョブエンドポイントガイド &#x200B;](./privacy-jobs.md) または [Privacy Service UI ユーザーガイド &#x200B;](../ui/user-guide.md) を参照してください。

Privacy Service エラーコードは、安定したパブリックコントラクトです。 各エラーコードは、プログラムによる処理と操作ワークフローで信頼できる失敗状態または完了状態を一意に識別します。

自動化を構築したりワークフローを監視したりする場合は、次の保証が適用されます。

* エラーコードは、公開後も安定しています。
* エラーメッセージは、わかりやすくするために変更される場合がありますが、コード値は変更されません。
* 新しいエラーコードは、時間の経過と共に追加される場合があります。既存のコードは再利用されません。

自動化や決定ロジックを実装するには、メッセージテキストではなく、エラーコードを使用します。 ポーリングやメッセージ文字列に依存せずにプライバシージョブを効率的に処理する方法、ジョブのステータスを監視する方法、およびエラーを処理する方法については、[Privacy Serviceのベストプラクティス &#x200B;](../best-practices.md) を参照してください。

## エラー応答形式 {#error-response-format}

Privacy Serviceは、ジョブとリクエストの応答でエラー情報を返します。 応答は、応答ペイロードに数値エラーコードと人間が判読できるメッセージを含む。

このエラーコードは、信頼できる結果を示しています。 メッセージには、トラブルシューティングに役立つ追加のコンテキストが表示されます。

このドキュメントでは、各エラーコードの意味と目的について説明します。 フィールドレベルの応答スキーマとリクエストの詳細については、[Privacy Service API ドキュメント &#x200B;](https://developer.adobe.com/experience-platform-apis/references/privacy-service/) を参照してください。

## エラードメイン {#error-domains}

エラーコードは、問題をより迅速に診断できるように、機能ドメイン別にグループ化されています。

このドキュメントで使用されるドメインは次のとおりです。

* **リクエストの検証**：リクエストの形式が正しくないか、無効な値が含まれています。 リクエスト構造と検証の要件については、[&#x200B; プライバシージョブエンドポイントガイド &#x200B;](./privacy-jobs.md) を参照してください。
* **認証とプロビジョニング**：組織またはユーザーが、必要なアクセス権を持っていません。 役割ベースの権限要件を確認するには、[&#x200B; 権限の管理 &#x200B;](../permissions.md) を参照してください。
* **ID と適用性**：識別子または名前空間はリクエストに適用されません。 サポートされる ID タイプと名前空間要件については、[&#x200B; プライバシーリクエストの ID データ &#x200B;](../identity-data.md) を参照してください。
* **レート制限**：送信ボリュームがプラットフォームの制限を超えています。 このエラーが発生した場合は、送信率を下げて、もう一度試してください。
* **データのアクセスと処理**：システムは、リクエストされたデータにアクセスしたり処理したりできません。 一般的な原因と修復手順については、[&#x200B; トラブルシューティングガイド &#x200B;](../troubleshooting-guide.md) を参照してください。
* **暗号化とキー管理**：必要な暗号化キーを使用できません。 鍵のアクセス、設定、回復のガイダンスについては、[&#x200B; 顧客管理キー &#x200B;](../../landing/governance-privacy-security/customer-managed-keys/overview.md) を参照してください。
* **ジョブの実行状態**：ジョブが完全、部分的に、または失敗して完了しました。 ジョブステータスカテゴリとその意味の説明については、[&#x200B; プライバシージョブエンドポイントガイド &#x200B;](./privacy-jobs.md#status-categories) を参照してください。

>[!NOTE]
>
>ドメインの割り当ては、内部のサービス境界ではなく、エラーコードの目的を反映しています。

## エラーコードの参照 {#error-code-reference}

次の表に、すべてのパブリック Privacy Service エラーコードを示します。

| エラーコード | HTTP ステータス | タイトル | 説明 |
| ---------- | ----------- | ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------| 
| 1000-400 | 400 | フォーマットエラー | 指定したアプリケーションの 1 つ以上のデータ値の形式に問題があります。 詳しくは、ジョブの詳細を確認してください。 |
| 1001-400 | 400 | 許可されていません | 組織がプロビジョニングされていません。 詳しくは、管理者にお問い合わせください。 |
| 1010-400 | 400 | 権限がありません | このアクションを実行するために必要な権限がありません。 管理者に連絡して、アクセス権をリクエストします。 |
| 1020-400 | 400 | アップロードとアーカイブの失敗 | アクセスデータのアップロード中またはアーカイブ中に問題が発生しました。 アクセスデータをアップロードして、もう一度試してください。 |
| 1021-400 | 400 | ジョブエラー | リクエストから作成された 1 つ以上のプライバシージョブが失敗しました。 詳細については、失敗したジョブの詳細を確認してください。 |
| 1022-400 | 400 | データアクセスの失敗 | 指定されたデータへのアクセス中に問題が発生しました。 追加情報については、ジョブの詳細を確認してください。 |
| 1023-400 | 400 | データアクセスの失敗 | 指定したデータセット ID へのアクセス中または特定中に問題が発生しました。 指定した ID が有効であることを確認してから、もう一度試してください。 |
| 1024-400 | 400 | 予期しないエラー | 予期しないエラーが発生しました。追加情報については、ジョブの詳細を確認してください。 |
| 1030-400 | 400 | ドキュメントレートの上限を超えました | ワークロードがドキュメントレートの上限を超えました。 送信率を下げてから、もう一度試してください。 |
| 1040-400 | 400 | キー暗号化のアクセス エラー | データストアが暗号化され、キーアクセスが取り消されたため、データを処理できませんでした。 カスタマーキーへのアクセスを復元するには、Key Vault 管理者にお問い合わせください。 |
| 6000-200 | 200 | 成功 | ジョブが正常に完了しました。 ジョブの詳細をレビューして、処理済みのレコードと結果を確認します。 |
| 6051-200 | 200 | プロビジョニングなし | お客様の組織は、リクエストされたアプリケーションに対してプロビジョニングされていません。 リクエストが適用されません。 |
| 6052-200 | 200 | ユーザー ID が見つかりません | 一部のユーザー ID が見つからず、不明なユーザー ID はこの製品には適用されません。 ユーザー ID が有効であることを確認してから、もう一度試してください。 |
| 6053-200 | 200 | 無効な名前空間 | 指定された ID 名前空間はこのアプリケーションには無効です。 システムで認識される名前空間を使用します。 |
| 6054-200 | 200 | 一部完了 | 適用可能なデータに対してジョブが完了しましたが、一部のデータがリクエストに適用できませんでした。 追加情報については、ジョブの詳細を確認してください。 |
