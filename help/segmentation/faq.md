# よくある質問

Adobe Experience Platform [!DNL Segmentation Service] には、セグメント定義や他のソースを使用してオーディエンスを作成できるユーザーインターフェイスおよび RESTful API が用意されています。 [!DNL Real-Time Customer Profile] データ。 これらのオーディエンスは、Platform 上で一元的に設定および管理され、任意のAdobeソリューションから簡単にアクセスできます。 以下は、オーディエンスとセグメントに関するよくある質問のリストです。

## Audience Portal と Audience Composition にアクセスできますか？

Audience Portal および Audience Composition は、Real-Time CDP Prime および Ultimate のすべてのお客様（B2C、B2B および B2P エディション）とJourney Optimizer Select、Prime、Ultimate Starter および Ultimate のお客様が利用できます。

この時点では、プロファイルベースのオーディエンスのみがサポートされます。 アカウントベースのオーディエンスのサポートは、今後のリリースで追加される予定です。

## 外部で生成された事前定義済みオーディエンスは、Audience Portal でサポートされますか？

はい、外部で生成された事前定義済みオーディエンスは、Audience Portal でサポートされます。 この時点で、CSV ファイルを使用して、外部で生成されたオーディエンスをインポートできます。 今後、バッチまたはストリーミングベースのソースコネクタを使用してオーディエンスを追加できるようになります。

## 外部で生成されたオーディエンスデータを、Platform の既存のプロファイルに紐付けることはできますか？

はい。主な識別子が一致する場合、外部で生成されたオーディエンスは、Platform の既存のプロファイルに結合されます。このデータを調整するには、最大 24 時間かかる場合があります。 プロファイルデータが存在しない場合は、データの取り込み時に新しいプロファイルが作成されます。

## 外部で生成されたオーディエンスを使用して、他のオーディエンスを構築することはできますか？

はい。外部で生成されたオーディエンスは、オーディエンスインベントリ内に表示され、 [セグメントビルダー](./ui/segment-builder.md).

## 外部的にアップロードされた属性をセグメント化の一部として使用することはできますか？

いいえ、できません。 プロファイル属性は、長時間持続する属性を意味します。一方、外部で生成されたオーディエンスデータは、外部で生成されたオーディエンスに関連付けられたコンテキストデータのみを含みます。

外部で生成されたオーディエンスのコンテキストデータ（エンリッチメント属性）は、 **not** ライフサイクルがアップロードされたオーディエンスに結び付けられているので、永続的に長く持続します。 その結果、一時的な性質上、これらのエンリッチメント属性は次のようになります **not** セグメント化で使用できます。

ただし、オーディエンスをバッチまたはファイルベースの宛先にマッピングする場合は、外部で生成されたこれらのエンリッチメント属性を使用して、オーディエンスを強化し、さらにダウンストリームのアクティベーションを強化できます。

この機能の詳細については、 [プロファイルの書き出し先をバッチ処理するためのオーディエンスデータのアクティブ化](../destinations/ui/activate-batch-profile-destinations.md#mapping).

## 外部で生成されたオーディエンスをAdobe Journey Optimizerに対してアクティブ化できますか？

この時点では、いいえ。 ただし、この機能は近い将来に使用可能になる予定です。

## 外部で生成されたオーディエンスを削除できますか？

この時点では、いいえ。 代わりに、このオーディエンスを非アクティブ化またはアーカイブできます。 この状態では、プロファイル **遺言** は、ダウンストリームアプリケーションで使用するためにアクティブなままになります。 外部で生成されたオーディエンスの削除のサポートは、今後のリリースで追加されます。

## Audience Portal と Audience Composition は、Real-Time CDP Partner Data のリリースとどのようにやり取りしますか？

Audience Portal と Audience Composition は、次の 2 つの方法でパートナーデータとやり取りします。

1. 見込み客プロファイルクラスとワークフローを使用してパートナー提供の見込み客リストを取り込む場合、見込み客は保持されます **別個に** プロファイルサービスの顧客プロファイルの結合から。 その結果、見込み客リストが **not** は、Audience Portal または Audience Composition のいずれかに表示されて使用されます。
2. パートナーが提供する属性を活用して **既存** ファーストパーティプロファイル、パートナーデータにエンリッチメントされたオーディエンス **遺言** は、Audience Portal と Audience Composition の両方で使用するために表示されます。

## 外部で生成されたオーディエンスを Audience Composition で使用することはできますか？

この時点では、いいえ。 ただし、この機能は近い将来に利用可能になるはずです。

## オーディエンス構成からすべてのダウンストリームの宛先およびチャネルにオーディエンスを送信できますか？

この時点では、いいえ。 現在、Adobe Journey Optimizerキャンペーンおよびリアルタイム CDP の宛先で、オーディエンスの構成のオーディエンスを使用できます。 Adobe Journey Optimizerジャーニーは、将来のリリースでサポートされる予定です。

## コンポジションの数にガードレールはありますか？

この時点では、 **10** サンドボックスごとにコンポジションを公開しました。 このガードレールは、今後のリリースで増やす予定です。

## Audience Composition のワークフローガードレールとは何ですか？

配置するコンポジションコンポーネントは、次のようにリジッドな構造に従います。

1. あなた **常に** 最初に [!UICONTROL 対象ユーザ] 開始アクティビティを選択するためのブロック。 最大で **1 つ** [!UICONTROL 対象ユーザ] ブロック
2. オプションで、 [!UICONTROL 除外] 続くブロック [!UICONTROL 対象ユーザ] ブロック
3. オプションで、 [!UICONTROL 強化] 続くブロック [!UICONTROL 除外] ブロック
4. オプションで、 [!UICONTROL ランク] または [!UICONTROL 分割] ブロック 以下が可能です。 **のみ** 1 組成当たり 1 つのブロックを持つ
5. あなた **常に** 終わりに [!UICONTROL 保存] ブロックを使用して、オーディエンスを保存します。

オーディエンスの構成の使用について詳しくは、 [Audience Composition UI ガイド](./ui/audience-composition.md).

## 別のコンポジションでオーディエンスコンポジションを使用することはできますか？

いいえ、オーディエンス構成を使用して作成されたオーディエンス **できません** は、別のオーディエンス構成の入力として使用されます。

## Audience Composition での分割はどのように機能しますか？

オーディエンス分割を使用すると、オーディエンスをより小さなグループに分割できます。 この分割は、グループ間の相互の排他性を強制します。 つまり、レコードが複数の分割パスの条件を満たす場合、レコードには **first** 左からのパスと **not** を他のパスに割り当てます。

分割ブロックの詳細については、 [Audience Composition UI ガイド](./ui/audience-composition.md#split).

## オーディエンス合成ワークフローですべてのセグメントタイプを使用できますか？

はい、すべてのセグメントタイプ ([バッチのセグメント化、ストリーミングのセグメント化、エッジのセグメント化](./home.md#evaluate-segments)) は、オーディエンス構成ワークフローでサポートされます。 ただし、現在、コンポジションは 1 日に 1 回だけ実行されるので、ストリーミングまたはエッジで評価されたオーディエンスが含まれている場合でも、コンポジションの実行時のオーディエンスのメンバーシップに基づいて実行されます。

## オーディエンスのプロファイルのメンバーシップを確認するにはどうすればよいですか？

プロファイルのオーディエンスのメンバーシップを確認するには、確認するプロファイルのプロファイルの詳細ページにアクセスします。 選択 **[!UICONTROL 属性]**&#x200B;に続いて **[!UICONTROL JSON を表示]**&#x200B;をクリックすると、 `segmentMembership` オブジェクトには、オーディエンスの ID が含まれます。
